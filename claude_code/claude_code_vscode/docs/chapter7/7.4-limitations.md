# 7.4 Claude 不是萬能的

Claude 很強大，但有些事情它做不到或不擅長。了解這些限制，才能更好地使用它。

## 🚫 限制 1：還是需要學習基礎知識

Claude 可以幫你寫程式，但：
- 你還是要懂基本的程式概念（變數、迴圈、函數）
- 你需要知道自己想要什麼
- 你要能判斷 Claude 的答案是否合理

### 比喻：
Claude 就像一個很會煮菜的廚師，但你還是要知道：
- 你想吃什麼（需求）
- 大概的做法（基本概念）
- 味道對不對（判斷能力）

## 🚫 限制 2：複雜邏輯需要人類思考

### Claude 適合：
- ✅ 實作已知的演算法
- ✅ 常見的設計模式
- ✅ 標準的功能

### Claude 較不適合：
- ❌ 全新的、創新的演算法設計
- ❌ 需要深度領域知識的複雜邏輯
- ❌ 高度客製化的商業邏輯

### 建議：
- 先自己思考整體架構
- 用 Claude 幫忙實作細節
- 複雜的邏輯自己設計，讓 Claude 協助優化

## 🚫 限制 3：安全性需要人工檢查

Claude 生成的程式碼可能有安全漏洞：
- SQL Injection（SQL 注入）
- XSS（跨站腳本攻擊）
- 敏感資訊洩漏
- 不安全的檔案操作

### 必做檢查：
```
✓ 檢查使用者輸入是否有驗證
✓ 密碼是否有加密
✓ API Key 是否寫死在程式碼中
✓ 檔案上傳是否有限制
✓ 資料庫查詢是否使用參數化查詢
```

## 🚫 限制 4：可能會「幻覺」（給出錯誤資訊）

有時 Claude 會：
- 引用不存在的函式庫
- 使用已棄用的語法
- 混淆不同版本的 API

### 防範方法：
- 重要的程式碼一定要測試
- 查閱官方文件確認
- 搜尋看看這個函式/方法是否真的存在

## 🚫 限制 5：不會取代你的思考

最重要的是：
- Claude 是**工具**，不是替代品
- 你的創意、判斷、決策無法被取代
- 學習的過程本身就是價值

## 正確心態

```
❌ 「有 Claude，我不用學程式了」
✅ 「用 Claude 加速學習，理解更深入」

❌ 「Claude 說的都是對的」
✅ 「Claude 提供參考，我要驗證和理解」

❌ 「直接複製 Claude 的程式碼」
✅ 「學習 Claude 的方法，內化成自己的能力」
```

## 最後的建議

### 🎯 Claude Code 的最佳用途：
- 學習新技術的得力助手
- 快速原型開發的加速器
- 程式碼品質改進的顧問
- 除錯過程的好夥伴

### 🎯 成為更好的開發者：
- 持續學習基礎知識
- 多寫、多練、多思考
- 參考 Claude，但不依賴
- 建立自己的理解和判斷力

記住：Claude Code 是來幫助你成長的，不是來取代你的。用對方法，你會進步神速！
