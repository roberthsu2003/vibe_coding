# 輕量化規格紀錄手冊

在 Vibe Coding 的流程中，「寫紀錄」不是為了交報告，而是為了給 AI 一個精確的「外部記憶體」。以下是讓你的專案不崩潰的最小化紀錄方案。

---

## 1. 核心檔案：`spec.md` 的威力

與其寫長篇大論，你只需要在專案根目錄建立一個 `spec.md`。

**為什麼一定要有它？**

- **對 AI 來說**：它是「防幻覺指南」。當對話變長，AI 會開始遺忘細節（例如：你之前決定蛇的初始長度是 3 還是 5），這份檔案能隨時把它拉回正軌。
- **對你來說**：它是「意圖備忘錄」。三週後你回來修改，不用重讀程式碼，看這份檔案就能知道「為什麼當初要設計牆壁可以穿透」。

---

## 2. 專案門面：`README.md`

`README.md` 是專案的進入點，讓「下一個接手的人」（或是另一個 AI Session）能快速進入狀況。

**建議包含內容：**

- **專案簡介**：一句話說明這是在做什麼。
- **啟動方式**：一條指令教人如何跑起來。
- **文檔導引**：明確指出「詳細細節請看 `spec.md`」。

💡 **`README.md` 實戰範本（以貪食蛇遊戲為例）：**

```markdown
# 經典貪食蛇 (Retro Snake Game)

這是一個基於瀏覽器的經典貪食蛇遊戲，旨在展示純 JavaScript 與 Canvas 的開發流程。

## 🛠️ 開發環境
- 使用 Vanilla JavaScript (ES6+)
- 無需安裝任何依賴，直接開啟即可執行

## 🚀 如何啟動
1. 下載本專案所有檔案。
2. 在瀏覽器中直接開啟 `index.html` 即可開始遊戲。

## 📖 規格說明
本專案的詳細技術細節、遊戲邏輯與未來待辦清單，請參考：
👉 [spec.md](./spec.md)
```

---

## 3. `spec.md` 實戰範本（以貪食蛇遊戲為例）

你可以直接複製以下結構到你的 `spec.md` 中。每次迭代完成一個功能（如：吃到食物長度增加），只要花 30 秒更新內容。

```markdown
# 專案規格紀錄 (spec.md) - 貪食蛇遊戲

## 🛠️ 技術棧 (Tech Stack)
- **語言**: HTML5, CSS3, JavaScript (Vanilla JS)
- **繪圖**: Canvas API (2D Context)
- **狀態管理**: 純物件與陣列 (無外部框架)
- **儲存**: localStorage (用於紀錄最高分)

## 📌 當前開發進度
- [x] 已完成：畫布渲染、蛇的基礎移動邏輯、食物隨機生成
- [ ] 進行中：碰撞偵測（牆壁與自身）
- [ ] 待辦：增加難度分級（隨得分增加速度）

## 🧠 核心邏輯 (The "Why")
- **蛇的身段紀錄**：使用座標陣列 `[{x, y}]`，頭部永遠是陣列第一個元素。
- **移動機制**：每一幀移除陣列末尾，並在頭部方向新增一個座標（除非吃到食物則不移除末尾）。
- **穿牆設定**：目前設定為「可穿牆」，蛇從左側消失會從右側出現，以增加新手友善度。

## 🛠️ 資料結構與 API
- **Snake Array**: `[{x: 10, y: 10}, {x: 10, y: 11}, ...]`
- **Game Object**: `{ score: 0, speed: 100, isPaused: false }`
- **localStorage Key**: `snake_high_score`

## ⚠️ 注意事項與副作用
- 畫布大小固定為 400x400，網格單位為 20x20，修改時需同步調整移動步長。
- 監聽 `keydown` 事件時須防止「快速反向切換」導致的自殺行為（例如：向上移動時立即按向下）。
```

---

## 🎉 總結：Vibe Coding 的高手準則

> **「動作即更新」**  
> 每當 AI 幫你寫出一個新的關鍵邏輯（例如：蛇吃到毒蘋果會縮短），請順手將它貼回 `spec.md`。這 30 秒的動作，能確保 AI 在下次對話中依然「神準」，並為未來的你省下數小時的除錯時間。
