# 輕量化規格紀錄手冊

在 Vibe Coding 的流程中，「寫紀錄」不是為了交報告，而是為了給 AI 一個精確的「外部記憶體」。本手冊說明如何用最少心力，記錄這三件事：

1. **專案說明**：整個專案在做什麼、給誰用、如何啟動  
2. **專案規格**：技術選型、核心邏輯、資料結構與注意事項  
3. **目前開發進度**：已完成、進行中、待辦項目  

以下用「兩份檔案」就能涵蓋上述所有內容。

---

## 一、專案說明：用 `README.md` 當專案門面

`README.md` 是專案的進入點，讓「下一個接手的人」或「下一個 AI Session」能快速理解**整個專案在做什麼**。

### 建議記錄內容

| 項目 | 說明 |
|------|------|
| **專案簡介** | 一句話說明這是在做什麼、目標對象是誰 |
| **啟動方式** | 一條指令或步驟，教人如何跑起來 |
| **規格與進度導引** | 明確指出「詳細規格與開發進度請看 `spec.md`」 |

### 範本（以貪食蛇遊戲為例）

```markdown
# 經典貪食蛇 (Retro Snake Game)

這是一個基於瀏覽器的經典貪食蛇遊戲，旨在展示純 JavaScript 與 Canvas 的開發流程。

## 🛠️ 開發環境
- 使用 Vanilla JavaScript (ES6+)
- 無需安裝任何依賴，直接開啟即可執行

## 🚀 如何啟動
1. 下載本專案所有檔案。
2. 在瀏覽器中直接開啟 `index.html` 即可開始遊戲。

## 📖 規格與進度
本專案的**技術規格**、**核心邏輯**與**目前開發進度**，請參考：
👉 [spec.md](./spec.md)
```

---

## 二、專案規格與開發進度：用 `spec.md` 集中記錄

在專案根目錄建立一個 `spec.md`，用來寫**專案規格**與**目前開發進度**。AI 與未來的你都能靠這份檔案快速回到正軌。

### 為什麼一定要有 spec.md？

- **對 AI**：是「防幻覺指南」。對話變長時 AI 容易遺忘細節（例如：蛇的初始長度是 3 還是 5），這份檔案能隨時把它拉回正軌。
- **對你**：是「意圖備忘錄」。過幾週再回來改，不用重讀程式碼，看這份檔案就能知道「為什麼當初要設計成可穿牆」。

### 建議記錄內容

| 類別 | 內容 |
|------|------|
| **專案規格** | 技術棧、核心邏輯與設計理由、資料結構與 API、注意事項與副作用 |
| **目前開發進度** | 已完成 ✓、進行中、待辦 |

---

## 三、`spec.md` 實戰範本（專案規格 ＋ 開發進度）

以下結構可直接複製到你的 `spec.md`。每完成一個功能，花 30 秒更新「目前開發進度」與相關規格即可。

```markdown
# 專案規格紀錄 (spec.md) - 貪食蛇遊戲

## 🛠️ 技術棧 (Tech Stack)
- **語言**: HTML5, CSS3, JavaScript (Vanilla JS)
- **繪圖**: Canvas API (2D Context)
- **狀態管理**: 純物件與陣列 (無外部框架)
- **儲存**: localStorage (用於紀錄最高分)

## 📌 目前開發進度
- [x] 已完成：畫布渲染、蛇的基礎移動邏輯、食物隨機生成
- [ ] 進行中：碰撞偵測（牆壁與自身）
- [ ] 待辦：增加難度分級（隨得分增加速度）

## 🧠 核心邏輯 (The "Why")
- **蛇的身段紀錄**：使用座標陣列 `[{x, y}]`，頭部永遠是陣列第一個元素。
- **移動機制**：每一幀移除陣列末尾，並在頭部方向新增一個座標（除非吃到食物則不移除末尾）。
- **穿牆設定**：目前設定為「可穿牆」，蛇從左側消失會從右側出現，以增加新手友善度。

## 🛠️ 資料結構與 API
- **Snake Array**: `[{x: 10, y: 10}, {x: 10, y: 11}, ...]`
- **Game Object**: `{ score: 0, speed: 100, isPaused: false }`
- **localStorage Key**: `snake_high_score`

## ⚠️ 注意事項與副作用
- 畫布大小固定為 400x400，網格單位為 20x20，修改時需同步調整移動步長。
- 監聽 `keydown` 事件時須防止「快速反向切換」導致的自殺行為（例如：向上移動時立即按向下）。
```

---

## 總結：三件事、兩份檔

| 要記錄的內容 | 放在哪裡 |
|--------------|----------|
| **專案說明**（在做什麼、如何啟動） | `README.md` |
| **專案規格**（技術、邏輯、資料結構、注意事項） | `spec.md` |
| **目前開發進度**（已完成 / 進行中 / 待辦） | `spec.md` |

> **「動作即更新」**  
> 每當 AI 幫你寫出一個新的關鍵邏輯（例如：蛇吃到毒蘋果會縮短），請順手將它貼回 `spec.md`，並更新「目前開發進度」。這 30 秒的動作，能確保 AI 在下次對話中依然「神準」，並為未來的你省下數小時的除錯時間。
