# 提示詞不知道怎麼寫？

面對空白輸入框，不知道該如何開口，或擔心寫得不夠準確？這篇以 **Vibe Coding**（用語感請 AI 寫程式）與**網頁小遊戲**為例，以下五種方式可以幫助你快速上手。

---

## 方式一：請 AI 直接提供 Prompt 範本

把你的目標告訴 AI，請它幫你產出一個可用的 prompt 範本，再依需求微調即可。

### 範例情境

你想請 AI 寫一個網頁版的「猜數字」小遊戲，但不知道該怎麼下指令。

**你可以這樣問：**

> 我想請 AI 幫我用 HTML + JavaScript 寫一個網頁版的猜數字遊戲（1～100），要有輸入框和送出按鈕，猜錯要提示「太高了」或「太低了」。請給我一個完整的 prompt 範本，讓我可以直接複製貼給 AI 用。

<details>
<summary>💡 參考解答（點擊展開）</summary>

<br>

AI 可能會回覆類似這樣的範本：
```
# 角色
你是一位熟悉前端開發的工程師，擅長用 HTML、CSS、JavaScript 做出互動網頁。

## 任務
請幫我寫一個網頁版的「猜數字」小遊戲。

### 遊戲規則（請依需求調整）
- 答案範圍：1～100 的隨機數字
- 玩家輸入數字後，系統提示「太高了」「太低了」或「猜對了」
- 需要：輸入框、送出按鈕、顯示提示訊息的區域

## 輸出格式
- 單一 HTML 檔案，內含 CSS 與 JavaScript
- 程式碼加上簡短中文註解
- 可直接用瀏覽器開啟執行
```

只要依你的遊戲規則微調「遊戲規則」區塊，再貼給 AI 即可產出程式碼。

</details>

## 方式二：請 AI 用一問一答的方式引導你

讓 AI 主動問你幾個問題，透過問答釐清需求，最後產出適合的 prompt。

### 範例情境

你想用 AI 做一個網頁版的「待辦清單」小應用，但不確定要寫哪些條件、按鈕或樣式。

**你可以這樣問：**

> 我想用 AI 寫一個網頁版的待辦清單（To-Do List），但不太確定怎麼寫 prompt。請用一問一答的方式引導我，問我幾個關鍵問題（例如：要哪些功能、要不要刪除/完成、介面風格等），再根據我的回答幫我產出完整的 prompt，讓 AI 可以幫我寫出程式碼。

<details>
<summary>💡 參考解答（點擊展開）</summary>

<br>

AI 可能會先問你：

1. **功能需求**：只要「新增」就好，還是也要「打勾完成」「刪除」？
2. **資料要不要保留**：重新整理頁面後清單要清空，還是用瀏覽器儲存（localStorage）記住？
3. **介面風格**：簡約白底、還是要有顏色區分已完成/未完成？
4. **輸出格式**：單一 HTML 檔，還是可以接受 HTML + CSS + JS 分開？

根據你的回答，AI 會組合出類似這樣的 prompt：
```
# 角色
你是熟悉前端開發的工程師，擅長用 HTML、CSS、JavaScript 做互動網頁。

## 任務
請用單一 HTML 檔案（含 CSS、JS）做一個待辦清單網頁。

### 功能
- 可輸入文字並新增一筆待辦
- 每一筆可「打勾完成」或「點擊刪除」
- （可選）用 localStorage 記住清單，重新整理後仍保留

## 輸出格式
程式碼加上簡短中文註解，可直接用瀏覽器開啟執行。
```

</details>

## 方式三：直接搜尋現成的 Prompt 庫

網路上有許多分類完整的 prompt 庫，可以直接搜尋、複製使用，省去從零開始的時間。

### 常用 Prompt 庫資源

| 資源 | 說明 | 連結 |
|------|------|------|
| **第五 AI 提示詞** | 10 萬+ 提示詞，涵蓋寫作、商業、開發等分類，一鍵複製 | [prompt.diwuai.com](https://prompt.diwuai.com/prompts) |
| **Prompt Library** | 開源專案，支援中文，可依分類瀏覽與搜尋 | [mrxie23.github.io/PromptLibrary](https://mrxie23.github.io/PromptLibrary/) |
| **Studio Global** | 100+ 專家製作提示，支援繁體中文，可團隊共享 | [studioglobal.ai](https://www.studioglobal.ai/zh-hant/prompt-library/) |
| **PromptVault** | 500+ 模板，適合開發者、內容創作者、小型企業 | 直接搜尋「PromptVault」 |

### 使用步驟

1. 打開任一 prompt 庫，在搜尋欄輸入關鍵字（如「網頁遊戲」「小遊戲」「待辦清單」「前端」）
2. 瀏覽結果，點選符合需求的 prompt
3. 一鍵複製，貼到 AI 對話框試用
4. 依實際需求微調（例如改遊戲規則、按鈕文字、配色）

<details>
<summary>💡 從庫中找到的「網頁小遊戲」prompt 範例（點擊展開）</summary>

<br>

```
請用 HTML + CSS + JavaScript 幫我寫一個網頁版小遊戲，要求：
1. 單一 HTML 檔案，內含所有程式碼，可直接用瀏覽器開啟
2. 遊戲類型：[例如：貪食蛇 / 猜數字 / 圈圈叉叉 / 打地鼠]
3. 介面簡潔，適合初學者閱讀與修改
4. 程式碼加上簡短中文註解

（可選）額外需求：例如「背景要黑色」「分數顯示在畫面上方」等。
```

把「遊戲類型」和「額外需求」替換成你想做的遊戲即可使用。

</details>

## 方式四：從一句簡單需求開始，請 AI 幫你擴充

先寫出最簡單的一句需求，再請 AI 幫你補齊細節和結構，逐步完善。

### 範例情境

你想請 AI 寫一個網頁版的「圈圈叉叉」（井字遊戲），但腦中只有「幫我寫一個圈圈叉叉遊戲」這幾個字。

**你可以這樣問：**

> 我想請 AI 幫我用網頁做一個圈圈叉叉（井字）遊戲，但我只想到「幫我寫一個圈圈叉叉遊戲」。請幫我把這句話擴充成一個完整、結構清楚的 prompt，包含角色、任務、遊戲規則、介面需求與輸出格式，讓 AI 可以產出可直接執行的 HTML 程式碼。

<details>
<summary>💡 參考解答（點擊展開）</summary>

<br>

AI 可能會產出：
```
# 角色
你是一位熟悉前端開發的工程師，擅長用 HTML、CSS、JavaScript 做互動網頁小遊戲。

## 任務
請用單一 HTML 檔案（含 CSS、JS）做一個經典的圈圈叉叉（井字）遊戲。

### 遊戲規則
- 3x3 九宮格，兩方輪流下（可先做雙人對戰，或玩家 vs 電腦）
- 先連成一直線（橫、直、斜）者獲勝
- 九格都下滿若無人獲勝則平手，需能顯示「重新開始」

### 介面需求
- 畫面上清楚顯示目前輪到誰、勝負結果
- 按鈕或點擊格子即可下棋，操作直覺

## 輸出格式
- 單一 HTML 檔案，可直接用瀏覽器開啟
- 程式碼加上簡短中文註解
```

之後可以依你想的玩法（例如改成單人對電腦、難度選項），在「遊戲規則」中增減項目。

</details>

## 方式五：參考類似範例，套用結構改寫

找到「目標相近」的 prompt 範例，保留其結構框架，把內容替換成你需要的版本。

### 範例情境

你想請 AI 寫一個網頁版的「打地鼠」小遊戲，手邊有「貪食蛇」的 prompt 可以參考。

**你可以這樣做：**

1. 打開貪食蛇的 prompt 範例（例如在「Prompt 的三大文字格式」那篇裡的 Markdown 範例）
2. 保留原有的結構（角色、任務、遊戲機制、介面設計、輸出格式）
3. 把「貪食蛇」相關的規則與介面，替換成「打地鼠」的內容

<details>
<summary>💡 改寫後的 prompt 範例（點擊展開）</summary>

<br>

```markdown
# 角色
你是一位精通互動遊戲開發的資深前端工程師。

## 任務
請使用單一 HTML 檔案（包含 CSS 與 JS），開發一個「打地鼠」小遊戲 (Whack-a-Mole)。

### 遊戲機制
- 畫面上有多個洞（例如 9 宮格），地鼠會隨機從洞中探出頭
- 玩家點擊地鼠即可得分，每打中一隻加分（例如 +10）
- 遊戲時間設定為 30 秒，時間到後顯示總分與「再玩一次」按鈕

### 介面設計
- 上方顯示剩餘時間與目前分數
- 地鼠出現/消失要有簡單動畫或效果，方便辨識
- 風格可可愛或復古，配色清楚即可

## 輸出格式
請提供一段可以直接執行的完整程式碼，並將邏輯加上中文註解。
```

**改寫重點**：角色、任務、結構框架不變，只替換成「打地鼠」的遊戲規則與介面描述。

</details>

## 快速選擇指南

| 你的狀況 | 建議方式 |
|----------|----------|
| 完全不知道從何開始 | ✅ 方式二：請 AI 一問一答引導 |
| 有目標但不會結構化 | ✅ 方式一：請 AI 給範本，或方式四：從簡單句擴充 |
| 想快速套用現成模板 | ✅ 方式三：搜尋 prompt 庫 |
| 手邊有類似小遊戲範例可參考 | ✅ 方式五：參考同類型改寫（例如貪食蛇→打地鼠） |

> 💡 **小提醒**：無論使用哪種方式，產出的 prompt 都建議先貼給 AI 試跑一兩次，再根據產出的程式碼效果微調。Vibe Coding 就是多試幾次，越用越順手！
