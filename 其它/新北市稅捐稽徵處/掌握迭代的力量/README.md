# 掌握迭代的力量

歡迎來到 Vibe Coding 的世界！在這裡，開發不再是枯燥的語法背誦，而是一場與 AI 之間的**「深度對話」**。

本章將帶你掌握 Vibe Coding 的核心引擎：**迭代 (Iteration)**。

---

## 1. 什麼是迭代？（核心引擎）

在傳統開發中，開發者追求「一次寫對」；但在 Vibe Coding，核心目標是**「讓程式長出來」**。

### 核心循環：Vibe Loop

不要試圖一次解決所有複雜問題，請遵循這個極速循環：

- **Prompt (下指令)**：用清晰的描述表達腦中的邏輯或畫面。
- **Generate (生成)**：觀察 AI 如何將想法轉化為程式碼。
- **Review (檢視)**：分析成果，確認哪些部分符合預期，哪些需要調整。
- **Refine (優化)**：針對落差處，引導 AI 進行修正。

> 💡 **Vibe 觀點**：保持開發的「手感 (Flow)」，接受過程中的不完美，透過連續對話逐步趨向理想結果。

---

## 2. 為什麼「迭代」是新手開發者的最強武器？

對於不具備深厚程式背景的人來說，迭代能有效克服三大開發障礙：

- ⚡ **即時回饋**：無需花費數月學習語法。在 Vibe Coding 中，第一行指令就能看見初步成果，建立開發信心。
- 🧠 **降低認知負載**：不需要同時處理整個系統的架構（如資料庫、前後端邏輯）。迭代允許開發者「一次專注於一個功能」。
- 🛡️ **極高容錯率**：AI 的容錯能力遠高於傳統編譯器。即便指令不夠精確，也能透過後續的微調來校正方向。

### ⚠️ 避免陷入「盲目抽卡」模式

若只是重複要求 AI 「重寫一次」而不去理解錯誤原因，開發將變成隨機的運氣遊戲。進階開發者的價值在於**「精準觀察」與「邏輯修正」**。

---

## 3. 實戰演練：打造「貪食蛇」遊戲

透過以下兩種模式的演練，可以快速體驗開發思維的轉變：

### 模式 A：指令追隨模式 (The Instruction Follower)

這適合初次嘗試，用來體驗 AI 的生成速度與邏輯理解能力。

1. **步驟 1**：建立基礎環境，畫出遊戲畫布與蛇的初始方塊。
2. **步驟 2**：加入鍵盤事件監聽，實現方向控制。
3. **步驟 3**：隨機產生食物，並實作碰撞偵測與蛇身增長邏輯。
4. **步驟 4**：定義結束條件（如撞牆或自撞），完成遊戲循環。

### 模式 B：目標導向模式 (The Problem Solver) 🌟 推薦挑戰

這是 Vibe Coding 的精髓。給定一個「最終目標圖」或「功能清單」，自行思考如何將其拆解為指令。

**核心心法**：
- **觀察現狀**：目前程式執行到了什麼程度？
- **定義落差**：現狀與最終目標還差哪些功能？
- **精準描述**：思考該如何下指令，才能引導 AI 補足缺失的邏輯。

---

## 4. 進階思考：直接載入「需求文件」的利弊

將所有功能寫成一份詳細的 `.md` 需求文件 (`Requirement.md`) 直接丟給 AI 雖然高效，但需注意以下風險：

- **結構化門檻**：若對開發邏輯不夠熟悉，難以寫出嚴謹且無衝突的需求描述。
- **除錯難度增加**：AI 若一次產出過多檔案或數百行程式碼，一旦出錯，定位問題的成本會大幅提升。

> 🎯 **實戰策略**：初期建議從「小步迭代」開始，透過對話引導 AI 逐步建構需求細節，而非一開始就要求完美的產出。
