# 新北市稅捐稽徵處 Vibe Coding 課程大綱

**總時數**：18 小時  
**天數**：3 天  
**每日**：2 節課，每節 3 小時  

**課程目的**：協助學員運用 AI 協作開發（Vibe Coding），完成 **3 個網頁小遊戲**。

---

## 課前準備（學員需先完成）

- 備妥 **Google 免費帳號**、**GitHub 免費帳號**
- 安裝或可使用的軟體：**Google AI Studio**（或可用的 AI 對話介面）、**VS Code**
- 可先瀏覽現有 Scratch 小遊戲作為參考：
  - [發票接物挑戰](https://scratch.mit.edu/projects/1190581926)
  - [使用牌照稅迷宮大冒險](https://scratch.mit.edu/projects/1202943790/)
  - [娛樂稅VS納保法記憶翻牌](https://scratch.mit.edu/projects/1202413885)

---

## 第一天：自然語言與 Prompt 基礎（6 小時）

### 第 1 節（3 小時）｜自然語言 + 迭代，產出第一個可玩原型

| 時間 | 內容 | 說明 |
|------|------|------|
| 0:00～0:30 | 課程說明與環境確認 | 說明 18 小時／3 天／3 個小遊戲目標；確認 Google、GitHub、VS Code、AI 介面可用。 |
| 0:30～1:15 | 自然語言 + 迭代的開發方法 | 講解「一句話描述目標 → AI 產出第一版 → 用迭代一步步完善」；對應教材：[自然語言+迭代的開發方法](../../自然語言+迭代的開發方法/README.md)。 |
| 1:15～1:30 | 休息 | — |
| 1:30～2:30 | 實作：第一個小遊戲原型 | 學員用自然語言請 AI 產出一個網頁小遊戲（如猜數字、貪食蛇、待辦清單等），在瀏覽器開啟並可操作。 |
| 2:30～3:00 | 迭代一輪：加一個功能並測試 | 每人針對剛產出的遊戲，用一句話請 AI 加一個功能（如顯示分數、再玩一次），測試後分享。 |

**本節產出**：每位學員至少有一個可在瀏覽器執行的網頁小遊戲原型。

---

### 第 2 節（3 小時）｜Prompt 格式與「不會寫怎麼辦」

| 時間 | 內容 | 說明 |
|------|------|------|
| 0:00～0:45 | Prompt 內容格式 | 介紹自然語言、YAML、Markdown 三種格式；ROSES 架構（Role、Objective、Steps、Examples、Scope）；對應教材：[Prompt 內容格式](prompt內容格式/README.md)。 |
| 0:45～1:00 | 休息 | — |
| 1:00～1:30 | 不會寫 Prompt 怎麼辦 | 五種方式：請 AI 給範本、一問一答引導、搜尋 Prompt 庫、從一句話擴充、參考範例改寫；對應教材：[不會寫 Prompt 怎麼辦](不會寫prompt怎麼辦/README.md)。 |
| 1:30～2:30 | 實作：用結構化 Prompt 改寫遊戲需求 | 學員選一個小遊戲（可延續第 1 節或重選），用 Markdown 或 ROSES 寫一段完整 Prompt，貼給 AI 產出或改寫程式。 |
| 2:30～3:00 | 分享與小結 | 兩人一組互相看對方 Prompt 與成果；講師講解常見問題與改進方式。 |

**本節產出**：學員能寫出較結構化的 Prompt，並產出或優化一個小遊戲。

---

## 第二天：規格紀錄與 PRD／Todo 開發流程（6 小時）

### 第 3 節（3 小時）｜輕量化規格紀錄 + 第二個遊戲規劃

| 時間 | 內容 | 說明 |
|------|------|------|
| 0:00～0:40 | 輕量化規格紀錄手冊 | 說明用 README.md（專案說明）與 spec.md（規格＋開發進度）記錄專案；對應教材：[輕量化規格紀錄手冊](輕量化規格紀錄手冊/README.md)。 |
| 0:40～0:50 | 休息 | — |
| 0:50～1:30 | 實作：為第一個遊戲寫 README + spec | 學員為昨日完成的遊戲建立 README.md（一句話說明、如何開啟）與 spec.md（技術棧、目前進度、核心邏輯或注意事項）。 |
| 1:30～2:50 | 第二個小遊戲：從 Prompt 到 PRD | 介紹「一份 Prompt 產出 PRD + todolist」；學員寫一份 Prompt 請 AI 產出 PRD.md 與 todolist.md，作為第二個遊戲的規劃；對應教材：[從一個 Prompt 到 PRD 與 Todo List](../../PRD/README.md)。 |
| 2:50～3:00 | 本節小結 | 確認每人都有第二個遊戲的 PRD 與 todolist，下一節依 todolist 開發。 |

**本節產出**：第一個遊戲有 README + spec；第二個遊戲有 PRD.md 與 todolist.md。

---

### 第 4 節（3 小時）｜依 Todo List 逐步開發（做一步、記錄、暫停、測試）

| 時間 | 內容 | 說明 |
|------|------|------|
| 0:00～0:25 | PRD／Todo 開發循環說明 | 講解「做 → 記錄 → 暫停 → 測試 → 矯正 → 下一步」；強調每步完成後要停下來測試，再進行下一項。 |
| 0:25～2:20 | 實作：依 todolist 開發第二個遊戲 | 學員依 todolist 一項一項請 AI 產出或修改程式；每完成一項在 todolist 打勾、暫停、在瀏覽器測試、有問題請 AI 修正後再下一項。講師巡堂協助。 |
| 2:20～2:35 | 休息 | — |
| 2:35～3:00 | 進度檢查與 spec 更新 | 每人更新第二個遊戲的 spec.md（目前開發進度）；若時間允許，可簡短分享目前做到哪一項、遇到什麼問題、如何用 Prompt 修正。 |

**本節產出**：第二個遊戲有明顯進度，且學員熟悉「依 todo 每步記錄、暫停測試」的節奏。

---

## 第三天：完成遊戲與部署（6 小時）

### 第 5 節（3 小時）｜第三個遊戲與除錯、規格維護

| 時間 | 內容 | 說明 |
|------|------|------|
| 0:00～0:30 | 複習與第三個遊戲選題 | 快速複習 PRD + todolist 流程；學員選定第三個小遊戲主題（可從 PRD 範例選：猜數字、記憶翻牌、貪食蛇、打磚塊、反應力測試等）。 |
| 0:30～1:00 | 產出第三個遊戲的 PRD + todolist | 學員寫 Prompt 請 AI 產出第三個遊戲的 PRD.md 與 todolist.md；或直接使用課程提供的 PRD 範例改寫。 |
| 1:00～1:15 | 休息 | — |
| 1:15～2:45 | 實作：開發第三個遊戲 + 回頭補齊前兩個 | 學員可選擇：專心完成第三個遊戲的 todolist，或回頭把第一、二個遊戲補齊／除錯。過程中持續更新各遊戲的 spec.md。 |
| 2:45～3:00 | 與 AI 除錯、Code Review 心法 | 講師示範：如何用自然語言描述「目前現象 vs 期望現象」請 AI 修正；如何請 AI 幫忙檢查程式碼或建議改進。 |

**本節產出**：三個遊戲都有可運作的版本，且 README／spec 有在更新。

---

### 第 6 節（3 小時）｜部署至 GitHub Pages + 成果分享

| 時間 | 內容 | 說明 |
|------|------|------|
| 0:00～0:50 | 部署至 GitHub Pages | 講解靜態網站部署：專案推送到 GitHub → Settings → Pages → 選分支與資料夾；對應教材：[Github 的部署](../../佈署/Github的部署/README.md)。學員將至少一個遊戲專案推上 GitHub 並開啟 Pages。 |
| 0:50～1:00 | 休息 | — |
| 1:00～2:00 | 學員完成部署與檢查 | 每人確認自己的 GitHub Pages 網址可正常開啟遊戲；若有問題（路徑、空白頁等），依教材常見問題排除或請講師協助。 |
| 2:00～2:40 | 成果分享 | 每人約 2～3 分鐘：展示 3 個遊戲（或已完成的部分）、分享一個「用 Prompt 解決問題」的經驗。 |
| 2:40～3:00 | 課程總結與後續建議 | 回顧三天：自然語言與迭代、Prompt 格式、規格紀錄、PRD＋Todo、部署；建議後續可練習的題目與可參考的教材連結。 |

**本節產出**：至少一個遊戲已部署至 GitHub Pages 並可透過網址開啟；學員完成成果分享。
