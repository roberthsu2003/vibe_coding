# 新北市稅捐稽徵處 Vibe Coding 課程大綱

**總時數**：18 小時  
**天數**：3 天  
**每日**：2 節課，每節 3 小時  

**課程目的**：學員運用 AI 協作開發（Vibe Coding），依講義完成課程提供的小遊戲範例，並將既有 Scratch 遊戲改為網頁版後部署至 GitHub Pages。

---

## 學員課前準備

學員需**已先瀏覽或玩過**以下現有 Scratch 小遊戲，作為後續選題與需求參考：

- [發票接物挑戰](https://scratch.mit.edu/projects/1190581926)
- [使用牌照稅迷宮大冒險](https://scratch.mit.edu/projects/1202943790/)
- [娛樂稅VS納保法記憶翻牌](https://scratch.mit.edu/projects/1202413885)

另請備妥 **Google 免費帳號**、**GitHub 免費帳號**，並可使用的 **Google AI Studio** 與 **VS Code**（第三天會使用）。

---

## 第一天：使用 Google AI Studio 完成課程小遊戲範例 + 講義基本操作

**本日環境**：全程使用 **Google AI Studio**。

**本日目標**：
- 依講義完成**課程提供的小遊戲範例**（由講師／教材指定之網頁小遊戲範例）。
- 學會講義中的**基本操作方式**，能自然與 AI 協作產出可執行的網頁。

**內容要點**：

- **自然語言 + 迭代的開發方法**  
  用一句話描述目標 → AI 產出第一版 → 用迭代一步步完善；對應教材：[自然語言+迭代的開發方法](../../自然語言+迭代的開發方法/README.md)。

- **Prompt 內容格式**  
  認識自然語言、YAML、Markdown 三種寫法與 ROSES 架構；對應教材：[Prompt 內容格式](prompt內容格式/README.md)。

- **不會寫 Prompt 怎麼辦**  
  五種方式：請 AI 給範本、一問一答引導、搜尋 Prompt 庫、從一句話擴充、參考範例改寫；對應教材：[不會寫 Prompt 怎麼辦](不會寫prompt怎麼辦/README.md)。

- **實作**  
  學員在 Google AI Studio 中依講義操作，完成課程提供的小遊戲範例（可複製產出的 HTML／程式到本機在瀏覽器測試）。

**本日產出**：學員能依講義在 Google AI Studio 完成指定的小遊戲範例，並熟悉自然語言、迭代與 Prompt 的基本操作。

---

## 第二天：使用 Google AI Studio + PRD 與 todolist 完成一個 Scratch 遊戲網頁版

**本日環境**：全程使用 **Google AI Studio**。

**本日目標**：
- 學員**任選一個**現有 Scratch 遊戲（發票接物挑戰、使用牌照稅迷宮大冒險、娛樂稅VS納保法記憶翻牌）。
- 以 **PRD（產品需求文件）** 與 **todolist.md（開發待辦清單）** 為開發方式，在 Google AI Studio 中產出該遊戲的網頁版。

**內容要點**：

- **從一個 Prompt 到 PRD 與 Todo List**  
  撰寫一份 Prompt 請 AI 產出 `PRD.md` 與 `todolist.md`；開發時依 todolist 一項一項實作，每完成一項就記錄、暫停、測試、必要時矯正再進行下一項；對應教材：[從一個 Prompt 到 PRD 與 Todo List](../../PRD/README.md)。

- **輕量化規格紀錄**  
  用 README.md 與 spec.md 記錄專案說明、規格與目前開發進度；對應教材：[輕量化規格紀錄手冊](輕量化規格紀錄手冊/README.md)。

- **實作**  
  學員在 Google AI Studio 中：  
  1. 選定一個 Scratch 遊戲，寫 Prompt 請 AI 產出該遊戲的 PRD.md 與 todolist.md。  
  2. 依 todolist 逐步請 AI 產出或修改程式，每步完成後記錄、測試。  
  3. 視需要維護 README／spec，讓專案狀態清楚。

**本日產出**：學員在 Google AI Studio 中完成「任選一個 Scratch 遊戲」的網頁版，且開發過程使用 PRD 與 todolist.md。

---

## 第三天：下載至 VS Code 並部署至 GitHub Pages

**本日環境**：**VS Code**、**Git**、**GitHub**。

**本日目標**：
- 將在 Google AI Studio 中完成的應用程式**下載／複製到本機**，在 **VS Code** 中開啟與管理。
- 將專案**部署至 GitHub Pages**，並學會兩種部署方式。

**內容要點**：

- **從 Google AI Studio 到 VS Code**  
  學員將第二天（或第一天）在 Google AI Studio 產出的程式碼儲存／下載到本機資料夾，用 VS Code 開啟專案，確認可在本機用瀏覽器開啟並正常執行。

- **部署至 GitHub Pages（兩種方式）**  
  對應教材：[Github 的部署](../../佈署/Github的部署/README.md)。  
  - **方式一：純靜態 HTML + CSS + JavaScript**  
    專案為單一或多個靜態檔（無建置步驟）。將檔案推送到 GitHub 倉庫後，於 Settings → Pages 選擇 **Deploy from a branch**，選定分支與資料夾（如根目錄或 `/docs`），由 GitHub 直接提供靜態檔案。  
  - **方式二：使用 GitHub Actions 部署**  
    專案需先建置（例如使用 Vite、React 等）時，在倉庫中設定 GitHub Actions 工作流程：建置完成後以 **deploy-pages** 等動作部署到 GitHub Pages；在 Settings → Pages 的 Source 選擇 **GitHub Actions**。

- **實作**  
  學員依自己專案類型（純靜態 或 需建置）選擇方式一或方式二，完成推送、設定與部署，並確認可透過 GitHub Pages 網址開啟遊戲。

**本日產出**：學員已將應用程式在 VS Code 中管理，並成功部署至 GitHub Pages（至少會依專案類型使用其中一種部署方式）。
