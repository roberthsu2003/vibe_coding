# 上午講義｜Cursor Vibe Coding 小遊戲

## 課程目標
- 認識 Vibe Coding 與 Cursor 的工作流程
- 完成一個自選主題的網頁小遊戲原型
- 練習以自然語言進行多輪提示與迭代

## 課程結構
1. Vibe Coding 概念導入（30 分）
2. Cursor 介面與 `CURSOR.md` 設定（30 分）
3. 小遊戲主題選擇與專案初始化（20 分）
4. Vibe Coding 實作迭代（80 分）
5. 成果分享與回饋（25 分）

## 先備準備
- 安裝並登入 Cursor
- 確認工作區資料夾已建立（建議：`vibe_coding/中華電信學院/小遊戲專案`）
- 下載或參考課程提供的 `CURSOR.md` 範例

## `CURSOR.md` 建議內容
```text
## Project Context
目標：使用 HTML/CSS/JavaScript 製作一款簡單的網頁小遊戲。
要求：程式碼需分檔清楚，提供基礎樣式與互動。請優先使用原生技術，必要時可引入輕量套件。

## Coding Style
- 使用現代 ES6 語法
- CSS 優先採用 Flexbox 或 Grid
- 重要函式需撰寫註解說明設計原因

## Tone
請以教學口吻回覆，解釋主要程式段落的用途。

## Goals
- 產生可執行的小遊戲原型
- 提供後續迭代建議
```

## 小遊戲主題 Prompt 範例
請依據自己的主題挑選合適的初始 Prompt，並在迭代階段嘗試加入更多功能。

### 記憶翻牌 Memory Match
- 初始 Prompt：
  「幫我用 HTML、CSS、JavaScript 建立一個記憶翻牌遊戲。畫面有 4x4 的卡片格子，隨機配對圖示。玩家翻開兩張牌時，比對是否相同，相同則保持翻開，不同則 1 秒後蓋回去。左上角顯示翻牌次數，遊戲結束顯示完成時間。」
- 迭代 Prompt：
  1. 「加入開始與重新開始按鈕，重新開始時洗牌並重置計時與翻牌次數。」
  2. 「新增難度選擇（3x4、4x4、4x5），並在畫面上顯示目前難度。」
  3. 「完成遊戲後顯示彈出視窗，記錄最佳時間，使用 localStorage 儲存。」

### 打磚塊 Brick Breaker
- 初始 Prompt：
  「使用 HTML Canvas 和原生 JavaScript 建一個打磚塊遊戲。包含球、板子、磚塊陣列。操作方式為左右方向鍵控制板子，球碰到磚塊後消失並得分，漏接球則扣一條生命，生命歸零時遊戲失敗。」
- 迭代 Prompt：
  1. 「在畫面右上角新增計分與剩餘生命顯示。」
  2. 「加入多關卡設計：完成一層磚塊後增加球速並生成新磚塊。」
  3. 「增加音效與粒子特效，打掉磚塊時播放音效與簡單動畫。」

### 反應力測試 Reaction Timer
- 初始 Prompt：
  「製作一個反應力測試網頁。畫面顯示一個按鈕，點擊『開始』後在 2~5 秒間隨機變色，變色後玩家需立刻按下，計算反應時間（毫秒）。顯示最近一次反應時間與歷史平均。」
- 迭代 Prompt：
  1. 「加入作弊防護：若玩家在變色前按下，顯示提醒並重新計時。」
  2. 「新增排行榜區塊，顯示 5 次最佳成績，使用 localStorage 儲存。」
  3. 「提供亮色與暗色主題切換，並記住使用者選擇。」

### 猜數字 Number Guess
- 初始 Prompt：
  「建立一個 1 到 100 猜數字的網頁遊戲。玩家輸入數字後顯示提示（太高或太低），記錄猜測次數，猜中後顯示恭喜訊息與總共花了幾次。」
- 迭代 Prompt：
  1. 「新增難度選擇：簡單（1-50）、普通（1-100）、困難（1-500），並根據難度決定可猜測次數上限。」
  2. 「加入提示按鈕，每局可以使用一次，提示範圍縮小至剩餘範圍的一半。」
  3. 「提供分享連結功能，完成後可以複製結果與難度資訊。」

### 貪食蛇 Snake Game
- 初始 Prompt：
  「請使用 HTML Canvas 與 JavaScript 建置貪食蛇遊戲。蛇可以用上下左右控制，吃到食物身體變長、得分增加，撞到牆或自己則遊戲結束，顯示最終得分。」
- 迭代 Prompt：
  1. 「加入速度調整：隨得分提高逐步增加移動速度。」
  2. 「增加暫停功能，按下空白鍵可以暫停與恢復。」
  3. 「新增排行榜與最佳成績紀錄，使用 localStorage 儲存並顯示前三名。」

## 實作流程建議
1. 在 Cursor 中建立專案資料夾並撰寫 `CURSOR.md`
2. 貼上初始 Prompt，由 AI 產出第一版程式碼
3. 實際執行並檢查未完成或錯誤的部分
4. 使用迭代 Prompt 補充功能、修正問題
5. 加入註解與簡單 README，整理學習過程

## 常見問題
- **AI 產出檔案缺漏**：提醒 AI 指定檔案名稱與內容格式
- **樣式或互動不如預期**：描述具體情境，請 AI 附上更新後的程式碼片段
- **想加入音效或素材**：請 AI 建議免版權資源，或提供占位圖示替代
