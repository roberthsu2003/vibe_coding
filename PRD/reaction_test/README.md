# 反應力測試遊戲 - 依步驟操作指南

> **技術說明**：本遊戲使用 HTML、CSS、JavaScript，**不需安裝任何環境**，在瀏覽器開啟 HTML 即可執行。

本頁說明如何依 [PRD 開發方式](../README.md)（一份 Prompt → 產出 PRD + todolist → 依步驟開發並每步暫停測試），完成反應力測試遊戲。請**依下列步驟順序**操作。

---

## 步驟總覽

| 階段 | 你要做的事 |
|------|------------|
| **步驟 A** | 用一份 Prompt 請 AI 產出 `PRD.md` 與 `todolist.md` |
| **步驟 B** | 依 todolist **一項一項**開發：每完成一項就**記錄、暫停、測試、矯正**，再進行下一項 |

---

## 步驟 A：產出 PRD 與 Todo List

1. 在專案目錄（例如 `reaction_test/`）建立一個要放程式與文件的資料夾。
2. 複製下方「**本遊戲的範例 Prompt**」，貼給 AI（如 Cursor、ChatGPT）。
3. 請 AI 產出兩份檔案內容：**PRD.md**、**todolist.md**。
4. 將 AI 產出的內容分別存成 `PRD.md` 與 `todolist.md`，放在你的專案目錄中。

### 本遊戲的範例 Prompt

```
我想做一個網頁版的「反應力測試」遊戲，用 HTML + CSS + JavaScript，單一 HTML 檔即可執行。
流程：準備好按空白鍵 → 隨機等待 1～5 秒 → 畫面變色或顯示「現在！」→ 玩家盡快再按空白鍵 → 顯示反應時間（毫秒）。
要有單次結果、最近幾次平均、評級（如極快/快/普通/慢），「再測一次」按鈕。

請幫我產生兩份文件：
1. PRD.md：產品需求文件，包含專案目標、目標使用者、功能清單、技術規格與驗收方式。
2. todolist.md：開發待辦清單，把功能拆成「一步一步可實作」的項目，每個項目要具體、可測試，
   並標註依賴關係（例如先有準備畫面與按鍵，再做隨機延遲與計時）。每完成一項就要能暫停讓使用者測試。
```

---

## 步驟 B：依 Todo List 一項一項開發（做完一步就記錄、就暫停）

請嚴格遵守下列**單步循環**，不要一次做完所有項目：

| 順序 | 你做什麼 |
|------|----------|
| 1. **做** | 打開 `todolist.md`，選定「下一個未完成的待辦」，把該項需求貼給 AI，請 AI 產出或修改程式。 |
| 2. **記錄** | 在 `todolist.md` 中把該項標記為已完成（打勾或寫完成日期）。 |
| 3. **暫停** | **不要立刻做下一項**，先停下來。 |
| 4. **測試** | 在瀏覽器開啟你的 HTML，實際操作剛完成的功能，確認是否符合預期。 |
| 5. **矯正** | 若有錯誤，用自然語言描述問題，請 AI 修正；修正後再測一次。 |
| 6. **下一步** | 確認沒問題後，再從 todolist 取下一個待辦，重複 1～6。 |

> 💡 **重點**：每完成一個待辦就**記錄並暫停測試**，有助於在開發過程中即時矯正程式，避免錯誤累積。

## 本遊戲的建議 Todo List（範例）

若 AI 產出的 todolist 太簡略或你想先有參考，可依下列範例建立或補充你的 `todolist.md`。學生可**依此順序**一項一項實作與測試：

| # | 待辦項目 | 完成後可測試的結果 |
|---|----------|--------------------|
| 1 | 建立單一 HTML，含「準備好後按空白鍵」提示與操作說明 | 可在瀏覽器看到說明並按空白鍵 |
| 2 | 按空白鍵後進入「等待」狀態，隨機 1～5 秒後將畫面變色或顯示「現在！」 | 等待時間不固定，會出現提示 |
| 3 | 提示出現時記錄開始時間；玩家再按空白鍵時記錄結束時間，計算並顯示反應時間（毫秒） | 能看到單次反應時間數字 |
| 4 | 依反應時間顯示評級（如 &lt;200ms 極快、200～300 快、300～500 普通、&gt;500 慢）並用顏色區分 | 結果有文字評級與顏色 |
| 5 | 顯示最近 10 次的反應時間列表與平均反應時間 | 可重複測幾次，看到列表與平均 |
| 6 | 「再測一次」按鈕：回到準備畫面，可重新開始一輪測試 | 可連續測多輪 |
| 7 | （進階）用 localStorage 儲存歷史記錄，重新開啟頁面仍能看到 | 紀錄可持久化 |
| 8 | （進階）難度選擇：簡單等待較長、困難較短，或提示較不明顯 | 主畫面可選難度 |

## PRD 範例參考

<details>
<summary>📄 點擊展開：反應力測試遊戲 PRD 範例</summary>

### 產品概述
- **產品名稱**：反應力測試遊戲（Reaction Test Game）
- **產品目標**：測量玩家從「提示出現」到「按下按鍵」的反應時間（毫秒）。
- **目標使用者**：所有年齡層。
- **技術要求**：HTML + CSS + JavaScript，單一 HTML 可執行，可選 localStorage。

### 功能需求摘要
- 準備 → 按鍵 → 隨機等待 → 提示出現（變色或文字）→ 玩家按鍵 → 顯示反應時間與評級；最近幾次列表與平均；再測一次；進階：難度、歷史記錄、圖表。

### 技術規格
- 使用 `performance.now()` 或 `Date.now()` 精確計時（毫秒）；鍵盤事件；隨機延遲 `Math.random()`；可選 localStorage、圖表。

</details>

## 給學生的建議

- **第一次**：依「步驟 A → 步驟 B」與「建議 Todo List」操作，習慣「做一步、測一步」的節奏。
- **熟練後**：可調整 todolist 或加入聽覺提示、連續測試模式、匯出資料等。
- **出錯時**：說明是「哪一步」、「目前現象」與「期望現象」，再請 AI 修正。
