# 記憶翻牌遊戲 - 依步驟操作指南

> **技術說明**：本遊戲使用 HTML、CSS、JavaScript，**不需安裝任何環境**，在瀏覽器開啟 HTML 即可執行。

本頁說明如何依 [PRD 開發方式](../README.md)（一份 Prompt → 產出 PRD + todolist → 依步驟開發並每步暫停測試），完成記憶翻牌遊戲。請**依下列步驟順序**操作。

---

## 步驟總覽

| 階段 | 你要做的事 |
|------|------------|
| **步驟 A** | 用一份 Prompt 請 AI 產出 `PRD.md` 與 `todolist.md` |
| **步驟 B** | 依 todolist **一項一項**開發：每完成一項就**記錄、暫停、測試、矯正**，再進行下一項 |

---

## 步驟 A：產出 PRD 與 Todo List

1. 在專案目錄（例如 `memory_card/`）建立一個要放程式與文件的資料夾。
2. 複製下方「**本遊戲的範例 Prompt**」，貼給 AI（如 Cursor、ChatGPT）。
3. 請 AI 產出兩份檔案內容：**PRD.md**、**todolist.md**。
4. 將 AI 產出的內容分別存成 `PRD.md` 與 `todolist.md`，放在你的專案目錄中。

### 本遊戲的範例 Prompt

```
我想做一個網頁版的「記憶翻牌」遊戲，用 HTML + CSS + JavaScript，單一 HTML 檔即可執行。
規則：4x4 共 16 張卡片（8 對），點擊翻開兩張，配對成功就保持翻開，配對失敗就翻回；
全部配對完成即過關，要有計分、翻牌次數、完成時間，「再玩一次」按鈕。

請幫我產生兩份文件：
1. PRD.md：產品需求文件，包含專案目標、目標使用者、功能清單、技術規格與驗收方式。
2. todolist.md：開發待辦清單，把功能拆成「一步一步可實作」的項目，每個項目要具體、可測試，
   並標註依賴關係（例如先有卡片網格與點擊翻開，再做配對判斷）。每完成一項就要能暫停讓使用者測試。
```

---

## 步驟 B：依 Todo List 一項一項開發（做完一步就記錄、就暫停）

請嚴格遵守下列**單步循環**，不要一次做完所有項目：

| 順序 | 你做什麼 |
|------|----------|
| 1. **做** | 打開 `todolist.md`，選定「下一個未完成的待辦」，把該項需求貼給 AI，請 AI 產出或修改程式。 |
| 2. **記錄** | 在 `todolist.md` 中把該項標記為已完成（打勾或寫完成日期）。 |
| 3. **暫停** | **不要立刻做下一項**，先停下來。 |
| 4. **測試** | 在瀏覽器開啟你的 HTML，實際操作剛完成的功能，確認是否符合預期。 |
| 5. **矯正** | 若有錯誤，用自然語言描述問題，請 AI 修正；修正後再測一次。 |
| 6. **下一步** | 確認沒問題後，再從 todolist 取下一個待辦，重複 1～6。 |

> 💡 **重點**：每完成一個待辦就**記錄並暫停測試**，有助於在開發過程中即時矯正程式，避免錯誤累積。

## 本遊戲的建議 Todo List（範例）

若 AI 產出的 todolist 太簡略或你想先有參考，可依下列範例建立或補充你的 `todolist.md`。學生可**依此順序**一項一項實作與測試：

| # | 待辦項目 | 完成後可測試的結果 |
|---|----------|--------------------|
| 1 | 建立單一 HTML，含 4x4 卡片網格（16 個可點擊元素），卡片顯示背面樣式 | 可在瀏覽器看到 16 張背面卡片 |
| 2 | 點擊卡片可「翻開」顯示正面（用 CSS 或 class 切換），一次可翻開兩張 | 點兩張會顯示正面內容 |
| 3 | 實作配對判斷：兩張相同則保持翻開，不同則約 1 秒後翻回背面 | 配對成功／失敗行為正確 |
| 4 | 全部 8 對都配對完成時顯示「遊戲完成」訊息 | 完成後有過關提示 |
| 5 | 計分：配對成功 +10、失敗 -1；畫面上方顯示當前分數與翻牌次數 | 分數與次數會隨操作變化 |
| 6 | 記錄並顯示完成時間（從開始到全部配對完成），格式 分:秒 | 完成時能看到耗時 |
| 7 | 「再玩一次」按鈕：重新洗牌、重置分數與時間、翻回全部背面 | 可重複玩多局 |
| 8 | （進階）卡片翻轉 CSS 動畫、配對成功／失敗的視覺回饋 | 翻牌與反饋有動畫 |
| 9 | （進階）用 localStorage 紀錄最佳成績（最高分或最短時間）並顯示 | 重新開啟頁面仍能看到紀錄 |

## PRD 範例參考

若你希望自己撰寫或微調 PRD，或 AI 未產出完整 PRD，可參考下方結構與內容。亦可將此範例提供給 AI，作為產出 `PRD.md` 的依據。

<details>
<summary>📄 點擊展開：記憶翻牌遊戲 PRD 範例</summary>

### 產品概述
- **產品名稱**：記憶翻牌遊戲（Memory Card Game）
- **產品目標**：考驗記憶力的卡片配對遊戲，玩家需記住位置並找出 8 對相同卡片。
- **目標使用者**：所有年齡層。
- **技術要求**：HTML + CSS + JavaScript，單一 HTML 可執行，可選 localStorage。

### 功能需求摘要
- 4x4 網格 16 張卡片（8 對）；點擊翻開、一次兩張；配對成功保持翻開、失敗延遲後翻回；計分、翻牌次數、完成時間；再玩一次；進階：難度（網格大小）、動畫、最佳成績。

### 技術規格
- HTML5 + CSS3（Flexbox/Grid、翻轉動畫）+ JavaScript（事件監聽、setTimeout 延遲翻回、localStorage）。

</details>

## 給學生的建議

- **第一次**：依「步驟 A → 步驟 B」與「建議 Todo List」操作，習慣「做一步、測一步」的節奏。
- **熟練後**：可調整 todolist（例如先做 4x4 再做 6x4）、或參考本頁 PRD 範例加入難度、動畫等。
- **出錯時**：說明是「哪一步」、「目前現象」與「期望現象」，再請 AI 修正。
