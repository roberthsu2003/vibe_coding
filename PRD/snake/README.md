# 貪食蛇遊戲 - 依步驟操作指南

> **技術說明**：本遊戲使用 HTML、CSS、JavaScript，**不需安裝任何環境**，在瀏覽器開啟 HTML 即可執行。建議使用 HTML5 Canvas 繪製遊戲畫面。

本頁說明如何依 [PRD 開發方式](../README.md)（一份 Prompt → 產出 PRD + todolist → 依步驟開發並每步暫停測試），完成貪食蛇遊戲。請**依下列步驟順序**操作。

---

## 步驟總覽

| 階段 | 你要做的事 |
|------|------------|
| **步驟 A** | 用一份 Prompt 請 AI 產出 `PRD.md` 與 `todolist.md` |
| **步驟 B** | 依 todolist **一項一項**開發：每完成一項就**記錄、暫停、測試、矯正**，再進行下一項 |

---

## 步驟 A：產出 PRD 與 Todo List

1. 在專案目錄（例如 `snake/`）建立一個要放程式與文件的資料夾。
2. 複製下方「**本遊戲的範例 Prompt**」，貼給 AI（如 Cursor、ChatGPT）。
3. 請 AI 產出兩份檔案內容：**PRD.md**、**todolist.md**。
4. 將 AI 產出的內容分別存成 `PRD.md` 與 `todolist.md`，放在你的專案目錄中。

### 本遊戲的範例 Prompt

```
我想做一個網頁版的「貪食蛇」遊戲，用 HTML + CSS + JavaScript，單一 HTML 檔即可執行，使用 HTML5 Canvas 繪製。
規則：蛇用方向鍵控制、持續移動，吃到食物變長、碰到牆或自己身體 Game Over；要顯示分數，Game Over 後可再玩一次，可選 localStorage 存最高分。

請幫我產生兩份文件：
1. PRD.md：產品需求文件，包含專案目標、目標使用者、功能清單、技術規格與驗收方式。
2. todolist.md：開發待辦清單，把功能拆成「一步一步可實作」的項目，每個項目要具體、可測試，
   並標註依賴關係（例如先有畫布與蛇的繪製，再做鍵盤控制與移動）。每完成一項就要能暫停讓使用者測試。
```

---

## 步驟 B：依 Todo List 一項一項開發（做完一步就記錄、就暫停）

請嚴格遵守下列**單步循環**，不要一次做完所有項目：

| 順序 | 你做什麼 |
|------|----------|
| 1. **做** | 打開 `todolist.md`，選定「下一個未完成的待辦」，把該項需求貼給 AI，請 AI 產出或修改程式。 |
| 2. **記錄** | 在 `todolist.md` 中把該項標記為已完成（打勾或寫完成日期）。 |
| 3. **暫停** | **不要立刻做下一項**，先停下來。 |
| 4. **測試** | 在瀏覽器開啟你的 HTML，實際操作剛完成的功能，確認是否符合預期。 |
| 5. **矯正** | 若有錯誤，用自然語言描述問題，請 AI 修正；修正後再測一次。 |
| 6. **下一步** | 確認沒問題後，再從 todolist 取下一個待辦，重複 1～6。 |

> 💡 **重點**：每完成一個待辦就**記錄並暫停測試**，有助於在開發過程中即時矯正程式，避免錯誤累積。

## 本遊戲的建議 Todo List（範例）

若 AI 產出的 todolist 太簡略或你想先有參考，可依下列範例建立或補充你的 `todolist.md`。學生可**依此順序**一項一項實作與測試：

| # | 待辦項目 | 完成後可測試的結果 |
|---|----------|--------------------|
| 1 | 建立單一 HTML，含 Canvas 與基本樣式，在畫布上繪製遊戲區域（網格或邊界） | 可在瀏覽器看到畫布與區域 |
| 2 | 用陣列儲存蛇的座標，在畫布上繪製蛇（頭與身體可同色），蛇靜止不動 | 畫面上看到一條蛇 |
| 3 | 實作鍵盤方向鍵控制蛇的「方向」，用 requestAnimationFrame 讓蛇依方向持續移動 | 蛇會依按鍵方向移動 |
| 4 | 在隨機位置生成食物（不落在蛇身上），繪製食物；蛇頭碰到食物時蛇身變長、分數增加、食物重新生成 | 吃到食物會變長與加分 |
| 5 | 碰撞偵測：蛇頭碰到邊界或自己身體時 Game Over，停止移動並顯示「Game Over」與分數 | 撞牆或自撞會結束並顯示訊息 |
| 6 | 開始畫面：標題、開始按鈕、操作說明；按開始後才進入遊戲 | 有明確的開始流程 |
| 7 | Game Over 後顯示「再玩一次」按鈕，點擊後重新開始（蛇與分數重置） | 可重複玩多局 |
| 8 | 畫面上方顯示當前分數（遊戲中與結束時） | 分數即時可見 |
| 9 | （進階）用 localStorage 儲存並顯示最高分 | 重新開啟頁面仍能看到最高分 |
| 10 | （進階）暫停鍵（如 P）、或隨分數增加速度 | 遊戲性進階調整 |

## PRD 範例參考

若你希望自己撰寫或微調 PRD，或 AI 未產出完整 PRD，可參考下方結構與內容。亦可將此範例提供給 AI，作為產出 `PRD.md` 的依據。

<details>
<summary>📄 點擊展開：貪食蛇遊戲 PRD 範例</summary>

### 產品概述
- **產品名稱**：貪食蛇遊戲（Snake Game）
- **產品目標**：經典街機遊戲，控制蛇吃食物成長，避免撞牆與自撞。
- **目標使用者**：所有年齡層。
- **技術要求**：HTML + CSS + JavaScript，HTML5 Canvas，單一 HTML 可執行，可選 localStorage。

### 功能需求摘要
- 蛇以陣列儲存座標、方向鍵控制、持續移動；食物隨機生成、吃到變長與加分；撞牆或自撞 Game Over；計分、開始畫面、再玩一次；進階：生命、速度、特殊食物、關卡、最高分。

### 技術規格
- Canvas、requestAnimationFrame、鍵盤事件、陣列操作、座標碰撞檢測、Math.random()、可選 localStorage。

</details>

## 給學生的建議

- **第一次**：依「步驟 A → 步驟 B」與「建議 Todo List」操作，先完成到「再玩一次」與「分數顯示」即可。
- **熟練後**：可加入生命、難度、特殊食物、關卡等，並對應更新 todolist。
- **出錯時**：說明是「哪一步」、「目前現象」與「期望現象」（例如「蛇不會轉彎」「吃到食物沒變長」），再請 AI 修正。
