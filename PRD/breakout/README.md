# 打磚塊遊戲 - 依步驟操作指南

> **技術說明**：本遊戲使用 HTML、CSS、JavaScript，**不需安裝任何環境**，在瀏覽器開啟 HTML 即可執行。建議使用 HTML5 Canvas 繪製遊戲畫面。

本頁說明如何依 [PRD 開發方式](../README.md)（一份 Prompt → 產出 PRD + todolist → 依步驟開發並每步暫停測試），完成打磚塊遊戲。請**依下列步驟順序**操作。

---

## 步驟總覽

| 階段 | 你要做的事 |
|------|------------|
| **步驟 A** | 用一份 Prompt 請 AI 產出 `PRD.md` 與 `todolist.md` |
| **步驟 B** | 依 todolist **一項一項**開發：每完成一項就**記錄、暫停、測試、矯正**，再進行下一項 |

---

## 步驟 A：產出 PRD 與 Todo List

1. 在專案目錄（例如 `breakout/`）建立一個要放程式與文件的資料夾。
2. 複製下方「**本遊戲的範例 Prompt**」，貼給 AI（如 Cursor、ChatGPT）。
3. 請 AI 產出兩份檔案內容：**PRD.md**、**todolist.md**。
4. 將 AI 產出的內容分別存成 `PRD.md` 與 `todolist.md`，放在你的專案目錄中。

### 本遊戲的範例 Prompt

```
我想做一個網頁版的「打磚塊」遊戲，用 HTML + CSS + JavaScript，單一 HTML 檔即可執行，使用 HTML5 Canvas 繪製。
規則：玩家控制平台反彈球，球擊破上方磚塊；球掉到底部失去生命，生命歸零 Game Over；要顯示分數與生命，不同排磚塊可不同分數，Game Over 或過關後可再玩一次。

請幫我產生兩份文件：
1. PRD.md：產品需求文件，包含專案目標、目標使用者、功能清單、技術規格與驗收方式。
2. todolist.md：開發待辦清單，把功能拆成「一步一步可實作」的項目，每個項目要具體、可測試，
   並標註依賴關係（例如先有畫布、平台與球，再做碰撞與磚塊）。每完成一項就要能暫停讓使用者測試。
```

---

## 步驟 B：依 Todo List 一項一項開發（做完一步就記錄、就暫停）

請嚴格遵守下列**單步循環**，不要一次做完所有項目：

| 順序 | 你做什麼 |
|------|----------|
| 1. **做** | 打開 `todolist.md`，選定「下一個未完成的待辦」，把該項需求貼給 AI，請 AI 產出或修改程式。 |
| 2. **記錄** | 在 `todolist.md` 中把該項標記為已完成（打勾或寫完成日期）。 |
| 3. **暫停** | **不要立刻做下一項**，先停下來。 |
| 4. **測試** | 在瀏覽器開啟你的 HTML，實際操作剛完成的功能，確認是否符合預期。 |
| 5. **矯正** | 若有錯誤，用自然語言描述問題，請 AI 修正；修正後再測一次。 |
| 6. **下一步** | 確認沒問題後，再從 todolist 取下一個待辦，重複 1～6。 |

> 💡 **重點**：每完成一個待辦就**記錄並暫停測試**，有助於在開發過程中即時矯正程式，避免錯誤累積。

## 本遊戲的建議 Todo List（範例）

若 AI 產出的 todolist 太簡略或你想先有參考，可依下列範例建立或補充你的 `todolist.md`。學生可**依此順序**一項一項實作與測試：

| # | 待辦項目 | 完成後可測試的結果 |
|---|----------|--------------------|
| 1 | 建立單一 HTML，含 Canvas，繪製遊戲區域（邊界）與底部平台（長條） | 可在瀏覽器看到畫布與平台 |
| 2 | 平台可依左右方向鍵（或滑鼠）移動，不超出邊界 | 平台可左右移動 |
| 3 | 在平台上方繪製一顆球，球有速度向量，用 requestAnimationFrame 讓球移動並碰到左右上邊界時反彈 | 球會動並反彈邊界 |
| 4 | 球碰到平台時反彈（依擊中位置調整角度），球掉到底部時視為失去一球 | 球與平台碰撞、掉底有反應 |
| 5 | 畫面上方繪製多排磚塊（如 5 排 x 8 個），不同排可不同顏色 | 看到磚塊陣列 |
| 6 | 球碰到磚塊時磚塊消失、球反彈、分數增加（不同排可不同分數） | 擊破磚塊會消失並加分 |
| 7 | 生命數（如 3 條）：球掉底減 1，生命歸零 Game Over；顯示剩餘生命與分數 | 生命與 Game Over 正確 |
| 8 | 開始畫面：標題、開始按鈕、操作說明；按開始後球才開始移動 | 有明確的開始流程 |
| 9 | Game Over 或全部磚塊擊破時顯示結束／過關訊息與「再玩一次」按鈕 | 可重複玩多局 |
| 10 | （進階）暫停鍵（如 P）、或隨擊破數增加球速 | 遊戲性進階調整 |
| 11 | （進階）多關卡、堅固磚塊需擊中多次、localStorage 最高分 | 進階功能 |

## PRD 範例參考

若你希望自己撰寫或微調 PRD，或 AI 未產出完整 PRD，可參考下方結構與內容。亦可將此範例提供給 AI，作為產出 `PRD.md` 的依據。

<details>
<summary>📄 點擊展開：打磚塊遊戲 PRD 範例</summary>

### 產品概述
- **產品名稱**：打磚塊遊戲（Breakout）
- **產品目標**：經典街機遊戲，控制平台反彈球擊破磚塊。
- **目標使用者**：所有年齡層。
- **技術要求**：HTML + CSS + JavaScript，HTML5 Canvas，單一 HTML 可執行，可選 localStorage。

### 功能需求摘要
- 平台（方向鍵／滑鼠）、球（速度與反彈）、磚塊（多排多色、擊中消失）；碰撞檢測（球與邊界、平台、磚塊）；生命、計分、開始畫面、Game Over／過關、再玩一次；進階：暫停、球速、關卡、特殊磚塊、最高分。

### 技術規格
- Canvas、requestAnimationFrame、鍵盤／滑鼠事件、矩形與圓形碰撞檢測、向量反彈、可選 localStorage。

</details>

## 給學生的建議

- **第一次**：依「步驟 A → 步驟 B」與「建議 Todo List」操作，先完成到「擊破磚塊、生命、再玩一次」即可。
- **熟練後**：可加入多關卡、堅固磚塊、特殊效果、最高分等，並對應更新 todolist。
- **出錯時**：說明是「哪一步」、「目前現象」與「期望現象」（例如「球穿過平台」「磚塊不會消失」），再請 AI 修正。
