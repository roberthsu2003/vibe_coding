# 猜數字遊戲 - 依步驟操作指南

> **技術說明**：本遊戲使用 HTML、CSS、JavaScript，**不需安裝任何環境**，在瀏覽器開啟 HTML 即可執行。

本頁說明如何依 [PRD 開發方式](../README.md)（一份 Prompt → 產出 PRD + todolist → 依步驟開發並每步暫停測試），完成猜數字遊戲。請**依下列步驟順序**操作。

---

## 步驟總覽

| 階段 | 你要做的事 |
|------|------------|
| **步驟 A** | 用一份 Prompt 請 AI 產出 `PRD.md` 與 `todolist.md` |
| **步驟 B** | 依 todolist **一項一項**開發：每完成一項就**記錄、暫停、測試、矯正**，再進行下一項 |

---

## 步驟 A：產出 PRD 與 Todo List

1. 在專案目錄（例如 `guess_number/`）建立一個要放程式與文件的資料夾。
2. 複製下方「**本遊戲的範例 Prompt**」，貼給 AI（如 Cursor、ChatGPT）。
3. 請 AI 產出兩份檔案內容：**PRD.md**、**todolist.md**。
4. 將 AI 產出的內容分別存成 `PRD.md` 與 `todolist.md`，放在你的專案目錄中。

### 本遊戲的範例 Prompt

```
我想做一個網頁版的「猜數字」遊戲，用 HTML + CSS + JavaScript，單一 HTML 檔即可執行。
規則：電腦隨機產生 1～100 的數字，玩家輸入猜測，系統提示「太大了」「太小了」直到猜對；
要顯示剩餘機會、猜測歷史，遊戲結束後可再玩一次。

請幫我產生兩份文件：
1. PRD.md：產品需求文件，包含專案目標、目標使用者、功能清單、技術規格與驗收方式。
2. todolist.md：開發待辦清單，把功能拆成「一步一步可實作」的項目，每個項目要具體、可測試，
   並標註依賴關係（例如先有輸入框與按鈕，再做判斷邏輯）。每完成一項就要能暫停讓使用者測試。
```

---

## 步驟 B：依 Todo List 一項一項開發（做完一步就記錄、就暫停）

請嚴格遵守下列**單步循環**，不要一次做完所有項目：

| 順序 | 你做什麼 |
|------|----------|
| 1. **做** | 打開 `todolist.md`，選定「下一個未完成的待辦」，把該項需求貼給 AI，請 AI 產出或修改程式。 |
| 2. **記錄** | 在 `todolist.md` 中把該項標記為已完成（打勾或寫完成日期）。 |
| 3. **暫停** | **不要立刻做下一項**，先停下來。 |
| 4. **測試** | 在瀏覽器開啟你的 HTML，實際操作剛完成的功能，確認是否符合預期。 |
| 5. **矯正** | 若有錯誤，用自然語言描述問題，請 AI 修正；修正後再測一次。 |
| 6. **下一步** | 確認沒問題後，再從 todolist 取下一個待辦，重複 1～6。 |

> 💡 **重點**：每完成一個待辦就**記錄並暫停測試**，有助於在開發過程中即時矯正程式，避免錯誤累積。

## 本遊戲的建議 Todo List（範例）

若 AI 產出的 todolist 太簡略或你想先有參考，可依下列範例建立或補充你的 `todolist.md`。學生可**依此順序**一項一項實作與測試：

| # | 待辦項目 | 完成後可測試的結果 |
|---|----------|--------------------|
| 1 | 建立單一 HTML 檔，含標題、數字輸入框、「猜測」按鈕、顯示訊息的區域 | 可在瀏覽器打開頁面，看到介面 |
| 2 | 實作「電腦隨機產生 1～100 數字」與「按猜測時比對並顯示太大/太小/猜對」 | 輸入數字後會得到提示，猜對有訊息 |
| 3 | 顯示剩餘機會數（例如 7 次），每次猜錯減 1，機會用盡顯示正確答案並結束 | 可測試機會遞減與結束條件 |
| 4 | 顯示猜測歷史（已猜過的數字列表） | 畫面上能看到歷史記錄 |
| 5 | 遊戲結束後顯示「再玩一次」按鈕，點擊後重新開始（新隨機數、清空歷史） | 可重複玩多局 |
| 6 | （進階）難度選擇：簡單 1～50、中等 1～100、困難 1～200，並調整機會數 | 主畫面可選難度並依難度限制機會 |
| 7 | （進階）用 localStorage 紀錄最佳成績（最少猜測次數）並顯示 | 重新開啟頁面仍能看到最佳成績 |

## PRD 範例參考

若你希望自己撰寫或微調 PRD，或 AI 未產出完整 PRD，可參考下方結構與內容。亦可將此範例提供給 AI，作為產出 `PRD.md` 的依據。

<details>
<summary>📄 點擊展開：猜數字遊戲 PRD 範例</summary>

### 產品概述
- **產品名稱**：猜數字遊戲（Number Guessing Game）
- **產品目標**：玩家透過「太大／太小」提示猜出電腦隨機產生的數字。
- **目標使用者**：所有年齡層，適合初學者練習邏輯與輸入處理。
- **技術要求**：HTML + CSS + JavaScript，單一 HTML 可執行，可選 localStorage 存紀錄。

### 功能需求摘要
- 電腦隨機產生指定範圍數字（如 1～100）；玩家輸入猜測；提示「太大了」「太小了」「猜對了」。
- 剩餘機會、猜測歷史、遊戲結束後再玩一次；進階：難度選擇、最佳成績、localStorage。

### 技術規格
- 使用 `Math.random()` 產生隨機數；輸入驗證；狀態管理（未開始／進行中／結束）；可選 localStorage。

</details>

## 給學生的建議

- **第一次**：可完全依照「步驟 A → 步驟 B」與「建議 Todo List」操作，習慣「做一步、測一步」的節奏。
- **熟練後**：可調整 todolist 的細項或順序，或參考本頁「PRD 範例參考」加入更多功能（難度、統計、視覺化等）。
- **出錯時**：先確認是「哪一步」出問題，用自然語言描述「目前現象」與「期望現象」，再請 AI 修正。
