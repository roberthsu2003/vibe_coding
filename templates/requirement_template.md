#### 💡 如何找參考範例？

不知道去哪裡找靈感嗎？以下是推薦的管道：

| **管道**        | **適合找什麼**     | **範例網站**                         |
| ------------- | ------------- | -------------------------------- |
| **Google 搜尋** | 真實在運作的 App/網站 | 搜尋「todo app」「記帳 app」直接看產品        |
| **設計模板網站**    | UI 畫面、設計稿     | Dribbble、Behance、Figma Community |
| **UI 範例庫**    | 特定元件、功能的設計參考  | Mobbin、Screenlane、Page Flows     |

**針對初學者的建議**：

| **類型**  | **推薦**        | **原因**                              |
| ------- | ------------- | ----------------------------------- |
| 最簡單     | **Mobbin**    | 依功能分類（登入、購物車、設定頁...），學生很容易找到特定功能的截圖 |
| 次簡單     | **Dribbble**  | 搜尋關鍵字就有大量 UI 設計圖，但品質參差不齊            |
| 直接看實際產品 | **Google 搜尋** | 最真實，但學生需要自己判斷哪些功能值得參考               |

**使用技巧**：
1. 找 2-3 個類似的產品/設計
2. 觀察它們的共同特徵
3. 挑選一個最符合你需求的作為主要參考
4. 截圖或記錄連結，方便後續查閱

## 如何使用這份模板？

### 步驟 1: 填寫模板
複製這份模板，根據你的專案填寫各個欄位。不確定的部分可以先留空，稍後補充。

### 步驟 2: 請 AI 審核
將填寫好的需求規格丟給 AI，使用以下 Prompt:

```
請審核以下需求規格書，指出:
1. 哪些需求描述不夠清楚
2. 有沒有遺漏的重要資訊
3. 有沒有互相矛盾的需求
4. 建議的優化方向

[貼上你的需求規格]
```

### 步驟 3: 根據 AI 回饋修正
根據 AI 的建議修正需求規格，讓需求更清楚、完整。

### 步驟 4: 生成計畫文件
將完善的需求規格丟給 AI，請它生成 plan.md 和 task.md:

```
請根據上述需求規格，幫我生成:
1. plan.md (高階架構設計)
2. task.md (詳細實作步驟)
```

---

**提示**:
- 填寫需求時，盡量使用**具體的例子**而非抽象描述
- 好的範例: "使用者名稱必須是 3-20 個字元，只能包含英文字母和數字"
- 不好的範例: "使用者名稱要合理"
