# 打磚塊遊戲 - PRD 需求文件教學

> **重要提示**：本遊戲使用 HTML、CSS 和 JavaScript 開發，**不需要安裝任何環境或套件**，直接在瀏覽器中打開 HTML 檔案即可運行。

## 什麼是 PRD？

PRD（Product Requirements Document，產品需求文件）是用來描述產品功能和需求的文件。在軟體開發中，PRD 幫助開發者理解：
- 產品要做什麼
- 功能有哪些
- 使用者如何與產品互動
- 技術要求是什麼

## 如何使用 PRD 產生程式碼

### 步驟 1：撰寫 PRD
根據下面的 PRD 範例，撰寫你的遊戲需求文件。

### 步驟 2：將 PRD 提供給 AI
將完整的 PRD 複製給 AI（如 Cursor），並加上以下 prompt：

```
請根據以下 PRD（產品需求文件）來建立這個遊戲。

[貼上你的 PRD 內容]

請使用 HTML、CSS 和 JavaScript 實作，使用 HTML5 Canvas 來繪製遊戲畫面，所有程式碼要能在瀏覽器中直接運行，不需要安裝任何環境。
```

### 步驟 3：測試與調整
測試 AI 產生的程式碼，如有問題可修改 PRD 或直接告訴 AI 需要改進的地方。

## PRD 範例

### 產品概述

**產品名稱**：打磚塊遊戲（Breakout）

**產品目標**：建立一個經典的街機遊戲，玩家控制平台反彈球來擊破磚塊。

**目標使用者**：所有年齡層的遊戲玩家

**技術要求**：
- 使用 HTML、CSS、JavaScript
- 使用 HTML5 Canvas 繪製遊戲畫面
- 不需要安裝任何環境或套件
- 直接在瀏覽器中運行
- 使用 localStorage 儲存遊戲記錄

---

### 功能需求

#### 1. 遊戲元素

**平台（Paddle）**：
- 位於畫面底部
- 可以用左右方向鍵控制移動
- 可以用滑鼠控制移動（滑鼠移動時平台跟隨）
- 有固定寬度（例如 100 像素）
- 移動速度適中，不會太快或太慢

**球（Ball）**：
- 圓形，有固定大小（例如 10 像素半徑）
- 從平台上方開始移動
- 有移動速度和方向（使用 x, y 速度向量）
- 碰到邊界、平台、磚塊時會反彈

**磚塊（Bricks）**：
- 位於畫面上方
- 排列成多排（例如 5-8 排）
- 每排有多個磚塊（例如 8-10 個）
- 不同排可以使用不同顏色
- 被球擊中時消失

#### 2. 遊戲機制

**碰撞檢測**：
- 球碰到左右牆壁：水平速度反向
- 球碰到頂部牆壁：垂直速度反向
- 球碰到平台：根據擊中位置調整反彈角度
- 球碰到磚塊：磚塊消失，球反彈

**遊戲開始**：
- 顯示開始畫面
- 按空白鍵或點擊開始按鈕開始遊戲
- 球從平台上方開始移動

**遊戲結束條件**：
- 球掉到底部（失去一條生命）
- 生命歸零時遊戲結束
- 清除所有磚塊時過關（如果有關卡系統）

#### 3. 生命系統

- **初始生命**：3 條生命
- **生命顯示**：在畫面上顯示剩餘生命數（例如：❤️❤️❤️）
- **失去生命**：球掉到底部時失去一條生命
- **重新開始**：失去生命後，球重新從平台上方開始

#### 4. 計分系統

- **計分規則**：
  - 擊破不同顏色的磚塊獲得不同分數
  - 例如：第一排 10 分，第二排 20 分，依此類推
- **分數顯示**：在畫面上方顯示當前分數
- **最終分數**：遊戲結束時顯示最終分數

#### 5. 遊戲狀態管理

- **開始畫面**：顯示遊戲標題、開始按鈕、操作說明
- **遊戲中**：顯示分數、生命數
- **暫停功能**：按 P 鍵可以暫停/繼續遊戲
- **遊戲結束畫面**：
  - 顯示最終分數
  - 顯示「遊戲結束」訊息
  - 提供「重新開始」按鈕（按 R 鍵或點擊按鈕）

#### 6. 速度調整（進階）

- 隨著分數增加，球的速度逐漸加快
- 或每擊破 5 個磚塊，速度增加一次
- 速度有上限，不會無限加快

#### 7. 特殊磚塊（進階功能）

- **普通磚塊**：擊中一次就消失
- **堅固磚塊**：需要擊中 2-3 次才會消失，用不同顏色表示
- **特殊效果磚塊**：
  - 加分磚塊：擊中後獲得額外分數
  - 加速磚塊：擊中後球的速度暫時增加
  - 擴展磚塊：擊中後平台變寬（持續 10 秒）
  - 多球磚塊：擊中後產生額外的球

#### 8. 關卡系統（進階功能）

- **多個關卡**：每個關卡有不同的磚塊排列
- **關卡難度**：關卡難度逐漸增加
- **關卡進度**：顯示當前關卡編號和剩餘磚塊數
- **通關條件**：清除所有磚塊即可進入下一關

#### 9. 視覺效果

- **球移動軌跡**：可選的軌跡效果
- **磚塊擊破效果**：磚塊被擊破時的粒子效果
- **平台動畫**：平台移動的平滑動畫
- **背景設計**：使用漸層或圖片作為背景

#### 10. 遊戲記錄（進階功能）

- **最高分數**：記錄並顯示最高分數
- **最佳關卡**：記錄到達的最高關卡
- **使用 localStorage**：儲存在瀏覽器本地

---

### 使用者介面需求

#### 1. 開始畫面
- 遊戲標題
- 開始按鈕
- 操作說明（方向鍵控制、P 暫停等）

#### 2. 遊戲畫面
- 分數顯示（左上角）
- 生命數顯示（右上角）
- 關卡編號（如果有關卡系統）
- Canvas 遊戲區域（中央）

#### 3. 遊戲結束畫面
- 最終分數顯示
- 最高分數顯示
- 「重新開始」按鈕
- 「返回主選單」按鈕

---

### 技術規格

#### 1. 檔案結構
- 建議使用單一 HTML 檔案，或分為 HTML、CSS、JavaScript 三個檔案

#### 2. 技術要求
- **HTML5 Canvas**：用於繪製遊戲畫面
- **requestAnimationFrame**：實現遊戲動畫迴圈
- **鍵盤事件**：監聽方向鍵和功能鍵
- **滑鼠事件**：監聽滑鼠移動（可選）
- **碰撞檢測**：使用基本的矩形和圓形碰撞檢測
- **localStorage**：儲存遊戲記錄

#### 3. 物理模擬
- 使用簡單的向量運算處理球的速度和方向
- 反彈角度根據擊中位置計算

---

### 非功能需求

1. **效能**：遊戲應該以 60 FPS 流暢運行
2. **響應性**：控制應該即時響應，沒有延遲
3. **可玩性**：難度適中，不會太簡單或太困難

---

## PRD 使用提示

### 給學生的建議

1. **從基礎開始**：第一次可以只實作基本功能（平台、球、磚塊、碰撞）
2. **逐步增加**：熟悉後再增加生命系統、計分、關卡等功能
3. **調整參數**：可以調整球的速度、平台大小、磚塊排列等來改變遊戲體驗
4. **學習物理**：觀察如何實現碰撞檢測和反彈邏輯

---

## 與迭代開發的差異

- **迭代開發**：從簡單版本開始，逐步增加功能
- **PRD 開發**：先完整規劃需求，一次性或分階段完成所有功能

兩種方式各有優點，可以根據情況選擇使用！
