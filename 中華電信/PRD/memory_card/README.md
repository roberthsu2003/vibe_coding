# 記憶翻牌遊戲 - PRD 需求文件教學

> **重要提示**：本遊戲使用 HTML、CSS 和 JavaScript 開發，**不需要安裝任何環境或套件**，直接在瀏覽器中打開 HTML 檔案即可運行。

## 什麼是 PRD？

PRD（Product Requirements Document，產品需求文件）是用來描述產品功能和需求的文件。在軟體開發中，PRD 幫助開發者理解：
- 產品要做什麼
- 功能有哪些
- 使用者如何與產品互動
- 技術要求是什麼

## 如何使用 PRD 產生程式碼

### 步驟 1：撰寫 PRD
根據下面的 PRD 範例，撰寫你的遊戲需求文件。你可以：
- 完全使用範例 PRD
- 修改範例 PRD，加入自己的想法
- 參考範例格式，自己撰寫全新的 PRD

### 步驟 2：將 PRD 提供給 AI
將完整的 PRD 複製給 AI（如 Cursor），並加上以下 prompt：

```
請根據以下 PRD（產品需求文件）來建立這個遊戲。

[貼上你的 PRD 內容]

請使用 HTML、CSS 和 JavaScript 實作，所有程式碼要能在瀏覽器中直接運行，不需要安裝任何環境。
```

### 步驟 3：測試與調整
- 測試 AI 產生的程式碼
- 如果有問題或需要調整，可以修改 PRD 或直接告訴 AI 需要改進的地方

## PRD 範例

### 產品概述

**產品名稱**：記憶翻牌遊戲（Memory Card Game）

**產品目標**：建立一個考驗記憶力的卡片配對遊戲，玩家需要記住卡片位置並找出配對。

**目標使用者**：所有年齡層的遊戲玩家

**技術要求**：
- 使用 HTML、CSS、JavaScript
- 不需要安裝任何環境或套件
- 直接在瀏覽器中運行
- 使用 localStorage 儲存遊戲記錄

---

### 功能需求

#### 1. 遊戲區域
- **卡片網格**：4x4 網格，共 16 張卡片（8 對）
- **卡片設計**：
  - 卡片初始狀態為背面（隱藏）
  - 卡片正面顯示不同的圖案或顏色
  - 每對卡片有相同的圖案/顏色

#### 2. 遊戲機制
- **翻牌規則**：
  - 點擊卡片可以翻開
  - 一次只能翻開兩張卡片
  - 翻開的卡片會顯示正面內容
- **配對判斷**：
  - 如果兩張卡片配對成功，保持翻開狀態
  - 如果配對失敗，在短暫顯示（約 1 秒）後自動翻回背面
- **遊戲完成**：
  - 當所有卡片都配對成功時，顯示「遊戲完成」訊息
  - 提供「再玩一次」按鈕

#### 3. 計分系統
- **計分規則**：
  - 每次成功配對得 10 分
  - 每次配對失敗扣 1 分
- **顯示位置**：在遊戲畫面上方顯示當前分數

#### 4. 統計資訊
- **翻牌次數**：記錄並顯示總共翻牌的次數
- **完成時間**：
  - 記錄遊戲開始到完成的時間
  - 遊戲完成時顯示總時間（格式：分:秒）

#### 5. 難度選擇（進階功能）
- **難度選項**：
  - 簡單：4x4 網格（8 對卡片）
  - 中等：6x4 網格（12 對卡片）
  - 困難：6x6 網格（18 對卡片）
- **計分倍數**：
  - 簡單：x1
  - 中等：x1.5
  - 困難：x2

#### 6. 視覺效果
- **卡片翻轉動畫**：使用 CSS 動畫實現平滑的翻轉效果
- **配對成功效果**：配對成功時卡片閃爍或高亮
- **配對失敗效果**：配對失敗時卡片震動或變色提示
- **背景設計**：使用漸層或圖片作為背景

#### 7. 遊戲記錄（進階功能）
- **最佳成績**：
  - 最快完成時間
  - 最高分數
  - 最少翻牌次數
- **本地儲存**：使用 localStorage 儲存記錄
- **排行榜**：顯示前 10 名最佳成績（可選）

---

### 使用者介面需求

#### 1. 主畫面
- 遊戲標題
- 難度選擇按鈕（如果有實作難度功能）
- 開始遊戲按鈕

#### 2. 遊戲畫面
- 分數顯示區域（頂部）
- 翻牌次數顯示（頂部）
- 卡片網格（中央）
- 遊戲完成訊息（彈出視窗或覆蓋層）

#### 3. 完成畫面
- 顯示完成時間
- 顯示最終分數
- 顯示翻牌次數
- 「再玩一次」按鈕
- 「返回主選單」按鈕

---

### 技術規格

#### 1. 檔案結構
- 建議使用單一 HTML 檔案，包含：
  - HTML 結構
  - CSS 樣式（`<style>` 標籤內）
  - JavaScript 邏輯（`<script>` 標籤內）
- 或分為三個檔案：
  - `index.html`
  - `style.css`
  - `script.js`

#### 2. 技術要求
- **HTML5**：使用語義化標籤
- **CSS3**：
  - 使用 Flexbox 或 Grid 進行佈局
  - 使用 CSS 動畫實現翻轉效果
  - 響應式設計（可選）
- **JavaScript**：
  - 使用 ES6+ 語法
  - 使用事件監聽器處理點擊
  - 使用 localStorage API 儲存資料
  - 使用 `setTimeout` 處理延遲翻回

#### 3. 瀏覽器相容性
- 支援現代瀏覽器（Chrome、Firefox、Safari、Edge）
- 不需要額外的 polyfill 或套件

---

### 非功能需求

1. **效能**：遊戲應該流暢運行，沒有明顯延遲
2. **可用性**：介面清晰易懂，操作直觀
3. **可維護性**：程式碼結構清晰，有適當的註解

---

### 測試需求

1. **功能測試**：
   - 卡片可以正常翻開
   - 配對邏輯正確
   - 計分系統正確
   - 遊戲完成判斷正確

2. **邊界情況**：
   - 快速連續點擊的處理
   - 所有卡片配對完成後的狀態

---

## PRD 使用提示

### 給學生的建議

1. **從簡單開始**：第一次使用時，可以先使用完整的 PRD，了解 AI 如何根據需求產生程式碼

2. **逐步修改**：熟悉後，可以嘗試：
   - 修改卡片數量
   - 改變計分規則
   - 增加新功能

3. **學習觀察**：
   - 觀察 AI 如何理解 PRD
   - 學習如何撰寫清晰的需求
   - 了解需求與程式碼的對應關係

4. **迭代改進**：
   - 如果產生的程式碼不符合預期，可以：
     - 修改 PRD 使其更明確
     - 直接告訴 AI 需要調整的地方
     - 參考「迭代」資料夾的方式，逐步增加功能

---

## 與迭代開發的差異

- **迭代開發**：從簡單版本開始，逐步增加功能
- **PRD 開發**：先完整規劃需求，一次性或分階段完成所有功能

兩種方式各有優點，可以根據情況選擇使用！
