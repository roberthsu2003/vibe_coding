# 猜數字遊戲 - Prompt 參考

> **重要提示**：本遊戲使用 HTML、CSS 和 JavaScript 開發，**不需要安裝任何環境或套件**，直接在瀏覽器中打開 HTML 檔案即可運行。

## 遊戲簡介
猜數字遊戲是一個經典的邏輯推理遊戲。電腦隨機產生一個數字，玩家需要透過提示（太大、太小）來猜出正確的數字。

## 迭代階段 1：基礎版本

### Prompt 範例
```
請幫我建立一個猜數字遊戲，使用 HTML、CSS 和 JavaScript。
遊戲需要能在瀏覽器中直接運行，不需要安裝任何額外的環境或套件。

基本需求：
1. 建立一個 HTML 檔案，包含輸入框和按鈕
2. 使用 JavaScript 的 Math.random() 隨機產生一個 1-100 之間的數字
3. 玩家在輸入框中輸入猜測的數字，點擊按鈕提交
4. 根據玩家的猜測給予提示：
   - 如果猜的數字太大，顯示「太大了！」
   - 如果猜的數字太小，顯示「太小了！」
   - 如果猜對了，顯示「恭喜！猜對了！」
5. 記錄猜測次數
6. 猜對後顯示總共猜了幾次
7. 提供「再玩一次」按鈕重新開始遊戲

請使用清晰的程式碼結構，並加上註解說明。使用 CSS 美化介面。
```

## 迭代階段 2：增加難度選擇與範圍設定

### Prompt 範例
```
在現有的猜數字遊戲基礎上，請增加以下功能：

1. 難度選擇：
   - 簡單：1-50 的範圍，最多 10 次機會
   - 中等：1-100 的範圍，最多 7 次機會
   - 困難：1-200 的範圍，最多 5 次機會

2. 自訂範圍：
   - 允許玩家自訂數字範圍（例如：1-1000）
   - 根據範圍大小自動計算允許的猜測次數

3. 機會限制：
   - 如果猜測次數超過限制，遊戲結束並顯示正確答案
   - 顯示剩餘機會數

4. 遊戲統計：
   - 記錄最佳成績（最少猜測次數）
   - 記錄總遊戲次數
   - 顯示平均猜測次數
```

## 迭代階段 3：圖形化介面

### Prompt 範例
```
請為猜數字遊戲建立更美觀的圖形化介面，使用 HTML、CSS 和 JavaScript：

1. 遊戲介面設計：
   - 使用 CSS 設計遊戲標題和整體佈局
   - 顯示當前難度或範圍
   - 顯示剩餘機會數

2. 輸入介面：
   - 數字輸入框（使用 HTML input type="number"）
   - 「猜測」按鈕
   - 或使用 HTML range input（滑桿）來選擇數字

3. 提示顯示：
   - 清楚顯示「太大」或「太小」的提示（使用不同顏色）
   - 顯示猜測歷史（已猜過的數字列表，使用 ul/li 或 div）
   - 用視覺化方式顯示數字範圍的縮小過程（使用進度條或數線）

4. 遊戲狀態：
   - 開始畫面（選擇難度，使用按鈕或選單）
   - 遊戲中畫面
   - 遊戲結束畫面（顯示結果、最佳成績）
```

## 迭代階段 4：增加提示功能與策略建議

### Prompt 範例
```
請在猜數字遊戲中增加提示功能和策略建議：

1. 提示功能（可選）：
   - 「提示」按鈕：顯示可能的數字範圍
   - 「建議」按鈕：根據當前範圍建議一個最佳猜測數字（二分搜尋策略）

2. 猜測歷史：
   - 顯示所有猜過的數字
   - 用顏色標記：太大（紅色）、太小（藍色）
   - 顯示每次猜測後縮小的範圍

3. 策略教學：
   - 遊戲結束後顯示使用的策略分析
   - 建議最佳策略（二分搜尋法）
   - 顯示如果使用最佳策略需要幾次猜測

4. 視覺化範圍：
   - 用進度條或數線顯示當前可能的數字範圍
   - 標記已猜過的數字位置
```

## 迭代階段 5：多人模式與競賽功能

### Prompt 範例
```
請在猜數字遊戲中增加多人模式和競賽功能：

1. 多人輪流模式：
   - 兩個玩家輪流猜測
   - 先猜對的玩家獲勝
   - 記錄每個玩家的猜測次數

2. 競賽模式：
   - 多個玩家同時進行遊戲（各自猜自己的數字）
   - 比較誰用最少的次數猜對
   - 顯示排行榜

3. 電腦對手模式：
   - 玩家可以選擇與電腦對戰
   - 電腦使用最佳策略（二分搜尋）
   - 比較玩家和電腦的表現

4. 記錄與統計：
   - 記錄多人遊戲的勝負記錄
   - 顯示對戰歷史
   - 計算勝率
```

## 技術建議

- **HTML + CSS + JavaScript**：不需要安裝任何環境，直接在瀏覽器中打開 HTML 檔案即可運行
- 使用 **HTML input** 元素來處理使用者輸入
- 使用 **CSS** 來美化介面，可以使用 CSS Grid 或 Flexbox 來佈局
- 使用 **localStorage** 來儲存遊戲記錄和最佳成績
- 使用 **CSS 動畫** 來增加視覺效果

## 學習重點

1. **階段1**：學習隨機數生成、條件判斷、迴圈控制、輸入處理
2. **階段2**：學習參數化設計、範圍計算、遊戲設定
3. **階段3**：學習 GUI 設計、事件處理、視覺化顯示
4. **階段4**：學習演算法（二分搜尋）、策略分析、教學功能
5. **階段5**：學習多人遊戲設計、競賽系統、資料比較
