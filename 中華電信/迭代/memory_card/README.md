# 記憶翻牌遊戲 - Prompt 參考

> **重要提示**：本遊戲使用 HTML、CSS 和 JavaScript 開發，**不需要安裝任何環境或套件**，直接在瀏覽器中打開 HTML 檔案即可運行。

## 遊戲簡介
記憶翻牌遊戲（Memory Card Game）是一個考驗記憶力的遊戲。玩家需要翻開卡片，記住卡片的位置，然後找出配對的卡片。

## 迭代階段 1：基礎版本

### Prompt 範例
```
請幫我建立一個記憶翻牌遊戲，使用 HTML、CSS 和 JavaScript。
遊戲需要能在瀏覽器中直接運行，不需要安裝任何額外的環境或套件。

基本需求：
1. 建立一個 HTML 檔案，包含一個 4x4 的卡片網格（共16張卡片，8對）
2. 使用 CSS 設計卡片樣式，卡片初始狀態為背面（隱藏）
3. 使用 JavaScript 處理點擊事件，點擊卡片可以翻開顯示內容
4. 一次只能翻開兩張卡片
5. 如果兩張卡片配對成功，則保持翻開狀態
6. 如果配對失敗，則在短暫顯示後翻回背面
7. 當所有卡片都配對成功時，顯示「遊戲完成」訊息

請使用清晰的程式碼結構，並加上註解說明。所有程式碼都放在同一個 HTML 檔案中，或分為 HTML、CSS、JavaScript 三個檔案。
```

## 迭代階段 2：增加計分系統

### Prompt 範例
```
在現有的記憶翻牌遊戲基礎上，請增加以下功能：

1. 計分系統：
   - 每次成功配對得 10 分
   - 每次配對失敗扣 1 分
   - 在畫面上顯示當前分數

2. 翻牌次數統計：
   - 記錄總共翻牌的次數
   - 顯示在畫面上

3. 完成時間：
   - 記錄遊戲開始到完成的時間
   - 遊戲完成時顯示總時間
```

## 迭代階段 3：難度調整

### Prompt 範例
```
請在記憶翻牌遊戲中增加難度選擇功能：

1. 遊戲開始前顯示難度選擇選單：
   - 簡單：4x4 網格（8對卡片）
   - 中等：6x4 網格（12對卡片）
   - 困難：6x6 網格（18對卡片）

2. 根據選擇的難度調整遊戲區域大小和卡片數量

3. 不同難度可以設定不同的計分倍數（簡單x1，中等x1.5，困難x2）
```

## 迭代階段 4：視覺效果與音效

### Prompt 範例
```
請為記憶翻牌遊戲增加視覺效果和音效：

1. 視覺效果：
   - 卡片翻轉動畫（平滑的翻轉效果）
   - 配對成功時的閃爍效果
   - 配對失敗時的震動效果
   - 背景漸層或圖片

2. 音效（可選）：
   - 翻牌音效
   - 配對成功音效
   - 配對失敗音效
   - 遊戲完成音效

3. 卡片設計：
   - 使用不同的圖案或顏色作為卡片內容
   - 卡片背面設計要美觀
```

## 迭代階段 5：遊戲記錄與排行榜

### Prompt 範例
```
請在記憶翻牌遊戲中增加遊戲記錄功能：

1. 記錄最佳成績：
   - 最快完成時間
   - 最高分數
   - 最少翻牌次數

2. 本地排行榜：
   - 儲存前 10 名最佳成績
   - 顯示排行榜畫面
   - 記錄玩家名稱（可選）

3. 遊戲統計：
   - 總遊戲次數
   - 平均完成時間
   - 平均分數

請使用瀏覽器的 localStorage 來儲存這些資料，這樣資料會保存在本地，重新整理頁面後仍然存在。
```

## 技術建議

- **HTML + CSS + JavaScript**：不需要安裝任何環境，直接在瀏覽器中打開 HTML 檔案即可運行
- 使用 **localStorage** 來儲存遊戲記錄，不需要後端伺服器
- 可以使用 **CSS 動畫** 來實現卡片翻轉效果
- 使用 **DOM 操作** 來處理卡片點擊和狀態更新

## 學習重點

1. **階段1**：學習基本的遊戲迴圈、事件處理、狀態管理
2. **階段2**：學習資料追蹤與顯示
3. **階段3**：學習參數化設計與條件邏輯
4. **階段4**：學習動畫與多媒體處理
5. **階段5**：學習資料持久化與 localStorage 的使用
