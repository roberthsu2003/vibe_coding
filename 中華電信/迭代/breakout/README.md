# 打磚塊遊戲 - Prompt 參考

> **重要提示**：本遊戲使用 HTML、CSS 和 JavaScript 開發，**不需要安裝任何環境或套件**，直接在瀏覽器中打開 HTML 檔案即可運行。

## 遊戲簡介
打磚塊（Breakout）是一個經典的街機遊戲。玩家控制一個平台（paddle）來反彈球，擊破上方的磚塊。目標是清除所有磚塊。

## 迭代階段 1：基礎版本

### Prompt 範例
```
請幫我建立一個打磚塊遊戲，使用 HTML、CSS 和 JavaScript。
遊戲需要能在瀏覽器中直接運行，不需要安裝任何額外的環境或套件。

基本需求：
1. 建立一個 HTML 檔案，包含一個遊戲畫布區域（例如 800x600 像素）
2. 使用 HTML5 Canvas 或 div 元素來繪製遊戲畫面
3. 在底部建立一個可移動的平台（paddle），可以用左右方向鍵或滑鼠控制
4. 建立一個球，從平台上方開始移動
5. 使用 JavaScript 處理碰撞檢測，球會反彈：碰到牆壁、平台、或磚塊時會反彈
6. 在頂部建立一排或多排磚塊
7. 當球擊中磚塊時，磚塊消失
8. 當球掉到底部時，遊戲結束（或失去一條生命）

請使用清晰的程式碼結構，並加上註解說明。使用 requestAnimationFrame 來實現遊戲迴圈。
```

## 迭代階段 2：增加生命與計分系統

### Prompt 範例
```
在現有的打磚塊遊戲基礎上，請增加以下功能：

1. 生命系統：
   - 玩家有 3 條生命
   - 球掉到底部時失去一條生命
   - 生命歸零時遊戲結束
   - 在畫面上顯示剩餘生命數

2. 計分系統：
   - 擊破不同顏色的磚塊獲得不同分數
   - 在畫面上顯示當前分數
   - 遊戲結束時顯示最終分數

3. 遊戲狀態：
   - 開始畫面（按空白鍵開始）
   - 遊戲中
   - 遊戲結束畫面（顯示分數，按 R 重新開始）
```

## 迭代階段 3：增加不同類型的磚塊

### Prompt 範例
```
請在打磚塊遊戲中增加不同類型的磚塊：

1. 普通磚塊：擊中一次就消失
2. 堅固磚塊：需要擊中 2-3 次才會消失
3. 特殊磚塊：
   - 加分磚塊：擊中後獲得額外分數
   - 加速磚塊：擊中後球的速度增加
   - 擴展磚塊：擊中後平台變寬
   - 多球磚塊：擊中後產生額外的球

2. 用不同顏色或圖案來區分不同類型的磚塊

3. 特殊效果有持續時間限制（例如平台擴展效果持續 10 秒）
```

## 迭代階段 4：關卡系統

### Prompt 範例
```
請在打磚塊遊戲中增加關卡系統：

1. 多個關卡：
   - 每個關卡有不同的磚塊排列
   - 關卡難度逐漸增加（更多磚塊、更複雜的排列）
   - 完成一關後自動進入下一關

2. 關卡選擇：
   - 遊戲開始時可以選擇關卡
   - 或從第一關開始依序進行

3. 關卡進度顯示：
   - 顯示當前關卡編號
   - 顯示關卡完成進度（剩餘磚塊數）

4. 通關條件：
   - 清除所有磚塊即可進入下一關
   - 完成所有關卡後顯示「恭喜通關」
```

## 迭代階段 5：視覺效果與音效

### Prompt 範例
```
請為打磚塊遊戲增加視覺效果和音效：

1. 視覺效果：
   - 球移動的軌跡效果
   - 磚塊被擊破時的粒子效果
   - 平台移動的平滑動畫
   - 背景動畫或漸層效果

2. 音效：
   - 球擊中平台音效
   - 球擊中磚塊音效
   - 磚塊被擊破音效
   - 特殊效果觸發音效
   - 背景音樂（可選）

3. 畫面優化：
   - 更精美的磚塊設計
   - 平台和球的視覺效果
   - 分數和生命數的顯示樣式
```

## 技術建議

- **HTML + CSS + JavaScript + Canvas**：不需要安裝任何環境，直接在瀏覽器中打開 HTML 檔案即可運行
- 使用 **HTML5 Canvas** 來繪製遊戲畫面
- 使用 **requestAnimationFrame** 來實現流暢的遊戲動畫迴圈
- 使用基本的數學運算來處理碰撞檢測和反彈邏輯
- 使用 **localStorage** 來儲存遊戲記錄

## 學習重點

1. **階段1**：學習物理碰撞檢測、向量運算、遊戲迴圈
2. **階段2**：學習遊戲狀態管理、生命系統設計
3. **階段3**：學習物件導向設計（不同磚塊類型）、效果系統
4. **階段4**：學習關卡設計、資料結構（關卡資料儲存）
5. **階段5**：學習視覺效果實作、音效整合
