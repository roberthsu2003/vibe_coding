# 統計分析 - 迭代 Prompt 參考

> **重要提示**：本教學使用迭代方式，從簡單的數據整理開始，逐步完善成專業的統計分析報告。

## 使用情境

統計分析是辦公室常見的工作，包括：
- 數據整理與分析
- 報表製作
- 趨勢分析
- 圖表製作
- 數據視覺化

透過迭代方式，您可以：
- 快速整理數據
- 逐步深入分析
- 產生專業報表
- 製作視覺化圖表
- 提出數據見解

## 迭代階段 1：基礎版本

### 目標
整理數據並產生基本的統計摘要。

### Prompt 範例

#### 情境 A：銷售數據分析

```
請幫我分析這份銷售數據，提供基本的統計摘要：

數據內容：
[貼上數據或描述數據來源，例如：Excel 檔案、CSV 檔案、或數據描述]

請提供：
1. 數據的基本資訊：有多少筆資料、時間範圍、主要欄位
2. 基本統計量：總和、平均、最大值、最小值
3. 主要發現：最重要的 3-5 個發現

請用簡潔的方式呈現。
```

#### 情境 B：客戶滿意度調查

```
請幫我分析這份客戶滿意度調查數據：

數據內容：
[貼上調查數據或描述]

請提供：
1. 調查基本資訊：回收問卷數、回收率、調查時間
2. 滿意度分數：平均分數、各項目的分數
3. 主要發現：最重要的 3-5 個發現

請用簡潔的方式呈現。
```

#### 情境 C：網站流量分析

```
請幫我分析這份網站流量數據：

數據內容：
[貼上流量數據或描述]

請提供：
1. 流量基本資訊：總瀏覽量、訪客數、時間範圍
2. 流量趨勢：是否有成長或下降
3. 主要來源：流量主要來自哪裡
4. 主要發現：最重要的 3-5 個發現

請用簡潔的方式呈現。
```

## 迭代階段 2：增加細節

### 目標
在基礎分析上加入更多詳細的統計資訊。

### Prompt 範例

#### 延續情境 A：銷售數據分析

```
請在剛才的銷售數據分析基礎上，增加以下詳細分析：

1. 分類分析：
   - 按產品類別分析銷售額
   - 按地區分析銷售額
   - 按時間（月/季）分析銷售額

2. 比較分析：
   - 與去年同期比較
   - 與上個月比較
   - 各產品類別之間的比較

3. 排名分析：
   - 銷售額前 10 名的產品
   - 成長率最高的產品
   - 各區域的銷售排名

4. 趨勢分析：
   - 銷售額的月趨勢
   - 是否有季節性變化
   - 成長或下降的趨勢

5. 異常值分析：
   - 是否有異常的數據點
   - 異常值的原因分析

請用表格和條列式的方式呈現，讓數據更清晰。
```

#### 延續情境 B：客戶滿意度調查

```
請在剛才的客戶滿意度調查分析基礎上，增加以下詳細分析：

1. 細項分析：
   - 各項目的詳細分數和分布
   - 各項目的滿意度等級分布（非常滿意、滿意、普通、不滿意、非常不滿意）
   - 各項目之間的相關性

2. 分群分析：
   - 按客戶類型分析（新客戶 vs 老客戶）
   - 按產品類型分析
   - 按地區分析

3. 開放式意見分析：
   - 如果有開放式問題，分析主要意見
   - 歸納常見的讚美和抱怨
   - 提出改進建議

4. 比較分析：
   - 與上次調查比較
   - 與業界標準比較
   - 各項目之間的比較

5. 關鍵指標：
   - NPS（淨推薦值）計算
   - 整體滿意度指標
   - 各項目的重要性分析

請用結構化的方式呈現，包含表格和圖表說明。
```

## 迭代階段 3：優化表達

### 目標
改善數據呈現方式，讓分析更易讀、更有說服力。

### Prompt 範例

#### 延續情境 A：銷售數據分析

```
請優化剛才的銷售數據分析，讓它：

1. 更有故事性：
   - 用敘述的方式說明數據趨勢
   - 解釋數據背後的原因
   - 提出數據的意義和影響

2. 結構更清晰：
   - 使用明確的標題和子標題
   - 用表格呈現數據
   - 用條列式呈現重點發現

3. 視覺化建議：
   - 建議適合的圖表類型（長條圖、折線圖、圓餅圖等）
   - 說明每個圖表要呈現什麼資訊
   - 說明圖表的標題和標籤

4. 重點突出：
   - 標示最重要的發現
   - 用顏色或格式突出關鍵數據
   - 說明哪些數據需要特別注意

5. 語言更專業：
   - 使用專業的統計術語
   - 避免口語化表達
   - 保持客觀但積極的語氣

請讓這份分析報告更易讀、更有說服力。
```

#### 延續情境 B：客戶滿意度調查

```
請優化剛才的客戶滿意度調查分析，讓它：

1. 更有洞察力：
   - 不只是呈現數據，要解釋數據的意義
   - 找出數據之間的關聯性
   - 提出深層的見解

2. 結構更專業：
   - 使用標準的調查報告格式
   - 包含執行摘要、詳細分析、結論建議
   - 用適當的章節組織內容

3. 視覺化建議：
   - 建議適合的圖表類型
   - 說明如何呈現滿意度分數
   - 說明如何呈現趨勢和比較

4. 行動導向：
   - 不只是分析問題，要提出解決方案
   - 根據數據提出改進建議
   - 設定優先順序

5. 語言更精準：
   - 使用具體的數字和百分比
   - 避免模糊的表述
   - 用數據支持論點

請讓這份報告更有價值和行動力。
```

## 迭代階段 4：專業化

### 目標
加入專業統計方法、標準格式、圖表製作，符合專業分析報告標準。

### Prompt 範例

#### 延續情境 A：銷售數據分析

```
請將剛才的銷售數據分析專業化：

1. 使用標準分析報告格式：
   - 執行摘要：1-2 頁的報告重點
   - 數據來源與方法：說明數據來源和分析方法
   - 詳細分析：各項目的詳細分析
   - 結論與建議：總結和行動建議
   - 附錄：原始數據和詳細表格

2. 加入專業統計方法：
   - 描述性統計：平均數、中位數、標準差、變異數
   - 相關性分析：各變數之間的相關性
   - 趨勢分析：時間序列分析、成長率計算
   - 比較分析：t 檢定、ANOVA（如果適合）

3. 製作專業圖表：
   - 長條圖：比較不同類別的數據
   - 折線圖：呈現趨勢變化
   - 圓餅圖：呈現比例分布
   - 散點圖：呈現相關性
   - 熱力圖：呈現多維度數據

4. 使用專業術語：
   - 用「描述性統計」而不是「基本統計」
   - 用「相關性分析」而不是「關聯分析」
   - 用「時間序列分析」而不是「趨勢分析」

5. 加入統計檢定（如果適合）：
   - 說明統計顯著性
   - 說明信賴區間
   - 說明樣本大小和代表性

請讓這份分析報告符合專業統計分析標準。
```

#### 延續情境 B：客戶滿意度調查

```
請將剛才的客戶滿意度調查分析專業化：

1. 使用標準調查報告格式：
   - 調查方法與樣本：說明調查方法、樣本大小、回收率
   - 執行摘要：主要發現和建議
   - 詳細分析：各項目的詳細分析
   - 結論與建議：總結和改進建議
   - 附錄：問卷內容、原始數據

2. 加入專業統計方法：
   - 描述性統計：平均數、標準差、分布
   - 信度分析：Cronbach's Alpha（如果適合）
   - 相關性分析：各項目之間的相關性
   - 因素分析：如果有結構性問題
   - 迴歸分析：如果有預測需求

3. 計算專業指標：
   - NPS（淨推薦值）
   - CSI（客戶滿意度指數）
   - 各項目的重要性-滿意度分析

4. 製作專業圖表：
   - 滿意度分數的長條圖
   - 滿意度等級的圓餅圖
   - 趨勢的折線圖
   - 重要性-滿意度矩陣圖

5. 加入統計檢定：
   - 說明樣本的代表性
   - 說明統計誤差範圍
   - 說明信賴水準

請讓這份報告符合專業調查分析標準。
```

## 迭代階段 5：個人化

### 目標
根據決策者需求、公司文化、實際應用進行最後調整。

### Prompt 範例

#### 延續情境 A：銷售數據分析

```
請將剛才的銷售數據分析個人化：

1. 根據決策者需求調整：
   - 如果決策者重視 ROI，加強投資報酬分析
   - 如果決策者重視成長，加強成長趨勢分析
   - 如果決策者重視風險，加強風險指標分析

2. 根據公司策略調整：
   - 對齊公司的年度目標
   - 符合公司的發展方向
   - 支援公司的決策需求

3. 加入實際應用：
   - 提出具體的行動建議
   - 說明數據對業務的影響
   - 提供可執行的改進方案

4. 考慮實際情況：
   - 根據實際資源調整建議
   - 考慮實際執行的可行性
   - 加入實際的時程和預算考量

5. 加入個人見解：
   - 展現分析者的專業判斷
   - 加入對數據的深度解讀
   - 提出創新的見解和建議

請讓這份分析報告既專業又實用。
```

#### 延續情境 B：客戶滿意度調查

```
請將剛才的客戶滿意度調查分析個人化：

1. 根據決策者關注點調整：
   - 如果重視客戶體驗，加強體驗相關的分析
   - 如果重視成本，加強成本效益分析
   - 如果重視競爭力，加強與競爭對手比較

2. 根據公司文化調整：
   - 符合公司的價值觀
   - 對齊公司的服務理念
   - 支援公司的品牌形象

3. 加入實際改進計畫：
   - 提出具體的改進措施
   - 說明改進的優先順序
   - 提供改進的時程和資源需求

4. 考慮實際執行：
   - 根據實際預算調整建議
   - 根據實際人力調整計畫
   - 考慮實際的技術和資源限制

5. 加入深度見解：
   - 不只是呈現數據，要解釋為什麼
   - 找出根本原因
   - 提出創新的解決方案

請讓這份報告既有洞察力又可行。
```

## 特殊情境處理

### 情境 1：數據不完整或品質不佳

```
請幫我分析這份數據，但數據可能不完整或品質不佳：

1. 數據問題：[說明數據的問題，例如：缺失值、異常值、樣本不足]
2. 分析限制：說明這些問題對分析的影響
3. 處理方式：說明如何處理這些問題
4. 建議：建議如何改善數據品質

請誠實說明數據的限制，但盡可能提供有價值的分析。
```

### 情境 2：需要預測或預測分析

```
請在剛才的分析基礎上，加入預測分析：

1. 預測目標：[說明要預測什麼，例如：下個月的銷售額]
2. 預測方法：使用什麼方法進行預測
3. 預測結果：預測的數值和範圍
4. 預測假設：說明預測的假設條件
5. 風險說明：說明預測的不確定性

請提供合理的預測，並說明預測的限制。
```

### 情境 3：需要比較分析

```
請在剛才的分析基礎上，加入比較分析：

1. 比較對象：[說明要與什麼比較，例如：去年同期、競爭對手、目標]
2. 比較指標：用哪些指標進行比較
3. 比較結果：比較的發現
4. 差異分析：說明差異的原因
5. 改進建議：根據比較結果提出建議

請提供深入的比較分析，不只是數字比較。
```

## 使用技巧

### 1. 提供足夠的數據資訊

**不好的 Prompt**：
```
幫我分析這份數據
```

**好的 Prompt**：
```
請幫我分析這份銷售數據：
- 數據來源：2024 年 Q1 的銷售報表
- 數據格式：Excel 檔案，包含產品名稱、銷售額、銷售數量、地區、日期
- 分析目標：了解銷售趨勢、找出熱門產品、分析地區差異
- 需要產出：統計摘要、趨勢分析、圖表建議
```

### 2. 明確指出需要改進的地方

**不好的回饋**：
```
這個分析不夠深入
```

**好的回饋**：
```
這個分析的趨勢部分還不夠深入，請補充：
1. 各月份的詳細趨勢分析
2. 與去年同期的比較
3. 季節性變化的分析
4. 成長率的計算和解釋
5. 預測未來 3 個月的趨勢
```

### 3. 逐步迭代，不要一次要求太多

**不好的方式**：
```
幫我做一個完美的數據分析，要專業、詳細、有深度、有預測、
有比較、有圖表、有建議、還要符合公司需求...
```

**好的方式**：
```
階段 1：先整理數據並產生基本統計摘要
階段 2：加入詳細的分類和比較分析
階段 3：優化表達和結構
階段 4：加入專業統計方法和標準格式
階段 5：根據實際需求進行最後調整
```
