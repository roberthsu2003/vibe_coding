# Email 寫作 - 迭代 Prompt 參考

> **重要提示**：本教學使用迭代方式，從簡單的 Email 初稿開始，逐步完善成專業的商務郵件。

## 使用情境

Email 寫作是辦公室最常見的工作之一。透過迭代方式，您可以：
- 快速產生 Email 初稿
- 逐步調整語氣和內容
- 針對不同對象客製化
- 處理複雜或敏感的溝通情境

## 迭代階段 1：基礎版本

### 目標
產生一封包含基本資訊的 Email 初稿。

### Prompt 範例

#### 情境 A：會議邀請

```
請幫我寫一封會議邀請的 Email，包含以下資訊：

1. 收件人：客戶公司的專案經理
2. 目的：邀請參加下週的專案進度會議
3. 時間：下週三下午 2:00
4. 地點：我們公司會議室
5. 議程：討論專案進度和下一步計畫

請用專業但友善的語氣，包含基本的問候和結尾。
```

#### 情境 B：工作報告

```
請幫我寫一封工作報告的 Email，給我的主管：

1. 主題：本週工作進度報告
2. 時間範圍：本週（週一到週五）
3. 主要完成事項：
   - 完成客戶需求分析
   - 參與產品設計會議
   - 準備下週的簡報資料
4. 遇到的問題：無重大問題
5. 下週計畫：繼續進行產品開發

請用簡潔明瞭的方式呈現。
```

#### 情境 C：客戶溝通

```
請幫我寫一封給客戶的 Email：

1. 目的：回覆客戶關於產品功能的詢問
2. 客戶問題：詢問產品是否支援某項功能
3. 回覆內容：是的，我們支援這項功能，並且可以客製化
4. 下一步：建議安排一次線上會議詳細說明

請用專業且積極的語氣，展現我們對客戶需求的重視。
```

## 迭代階段 2：增加細節

### 目標
在基礎版本上加入更多具體細節，讓 Email 更完整。

### Prompt 範例

#### 延續情境 A：會議邀請

```
請在剛才的會議邀請 Email 基礎上，增加以下細節：

1. 會議預計時長：1.5 小時
2. 需要準備的資料：請攜帶上次會議的決議事項
3. 會議議程細節：
   - 前 30 分鐘：專案進度報告
   - 中間 45 分鐘：討論遇到的挑戰和解決方案
   - 最後 15 分鐘：確定下一步行動項目
4. 如果無法參加，請提前告知
5. 會議室位置：3 樓 301 會議室
6. 停車資訊：大樓地下停車場可免費停車

請保持專業友善的語氣，讓收件人清楚知道需要準備什麼。
```

#### 延續情境 B：工作報告

```
請在剛才的工作報告 Email 基礎上，增加以下細節：

1. 每個完成事項的具體成果：
   - 客戶需求分析：已完成 80% 的客戶訪談，整理出 15 項核心需求
   - 產品設計會議：參與 3 場會議，提出 5 個設計建議，其中 3 個被採納
   - 簡報資料：已完成初稿，包含 20 頁的內容
2. 遇到的問題細節：
   - 客戶需求分析時，部分客戶回覆較慢，已主動追蹤
3. 下週計畫的具體行動：
   - 完成剩餘 20% 的客戶訪談
   - 開始撰寫產品規格書
   - 準備並進行簡報演練

請用條列式呈現，讓主管能快速掌握重點。
```

## 迭代階段 3：優化表達

### 目標
改善語言表達，讓 Email 更流暢、更有說服力。

### Prompt 範例

#### 延續情境 A：會議邀請

```
請優化剛才的會議邀請 Email，讓它：

1. 開頭更吸引人：不要只是「您好」，加入一些個人化的問候
2. 說明會議的重要性：為什麼這次會議很重要
3. 語氣更親切：雖然專業，但不要太正式僵硬
4. 結構更清晰：使用適當的段落和標題
5. 結尾更有行動力：明確說明希望對方做什麼

請保持專業，但讓整體感覺更溫暖、更有說服力。
```

#### 延續情境 C：客戶溝通

```
請優化剛才給客戶的 Email，讓它：

1. 開頭表達感謝：感謝客戶的詢問和對我們產品的興趣
2. 更詳細說明功能：不只是說「支援」，要說明如何支援、有什麼優勢
3. 加入具體案例：如果有的話，提到其他客戶使用這個功能的成功經驗
4. 語氣更積極：展現我們很樂意協助客戶解決問題
5. 結尾更明確：清楚說明下一步，並提供聯絡方式

請讓客戶感受到我們的重視和專業。
```

## 迭代階段 4：專業化

### 目標
加入專業術語、格式規範，符合商務郵件標準。

### Prompt 範例

#### 延續情境 A：會議邀請

```
請將剛才的會議邀請 Email 專業化：

1. 加入正式的 Email 格式：
   - 明確的主旨行（Subject）
   - 適當的稱謂
   - 正式的結尾簽名檔
2. 使用專業術語：
   - 用「議程」而不是「討論內容」
   - 用「行動項目」而不是「要做的事」
3. 加入會議資訊的結構化呈現：
   - 使用表格或清晰的格式呈現時間、地點、議程
4. 加入後續追蹤：
   - 說明會後會提供會議記錄
   - 提供會議連結（如果是線上會議）
5. 符合公司郵件規範：
   - 包含公司資訊
   - 加入免責聲明（如果需要）

請讓這封 Email 看起來專業且完整。
```

#### 延續情境 B：工作報告

```
請將剛才的工作報告 Email 專業化：

1. 使用標準的工作報告格式：
   - 明確的報告標題和日期
   - 使用表格呈現數據
   - 使用圖表說明進度（如果適合）
2. 加入量化指標：
   - 完成度百分比
   - 時間花費
   - 預期完成時間
3. 使用專業的工作術語：
   - 用「里程碑」而不是「階段」
   - 用「風險評估」而不是「可能的問題」
4. 加入數據支持：
   - 如果有數據，用數字說話
   - 如果有比較，用百分比或趨勢說明
5. 結構化呈現：
   - 使用標題和子標題
   - 使用條列式呈現重點

請讓這份報告符合公司的工作報告標準。
```

## 迭代階段 5：個人化

### 目標
根據收件人、公司文化、個人風格進行最後調整。

### Prompt 範例

#### 延續情境 A：會議邀請

```
請將剛才的會議邀請 Email 個人化：

1. 根據收件人調整：
   - 如果收件人是長期合作的夥伴，語氣可以更親切
   - 如果收件人是新客戶，語氣要更正式但友善
   - 加入一些個人化的元素，例如提到上次見面的內容
2. 符合公司文化：
   - 如果公司文化較輕鬆，可以加入一些輕鬆的語氣
   - 如果公司文化較正式，保持專業正式
3. 加入個人風格：
   - 如果我是比較直接的人，Email 可以更簡潔
   - 如果我是比較細心的人，可以加入更多細節說明
4. 考慮文化差異：
   - 如果是跨國溝通，注意文化差異
   - 使用適當的稱謂和問候語
5. 加入情感連結：
   - 表達對這次會議的期待
   - 展現對合作的重視

請讓這封 Email 既專業又有人情味。
```

#### 延續情境 C：客戶溝通

```
請將剛才給客戶的 Email 個人化：

1. 根據客戶關係調整：
   - 如果是新客戶，強調我們的專業和服務
   - 如果是老客戶，可以更親切，提到過去的合作經驗
2. 根據客戶類型調整：
   - 如果是技術背景的客戶，可以加入更多技術細節
   - 如果是業務背景的客戶，強調商業價值
3. 加入個人化的服務承諾：
   - 表達我們會提供客製化的服務
   - 說明我們會指派專人協助
4. 考慮客戶的時區和語言：
   - 使用客戶熟悉的語言風格
   - 考慮時差，選擇適當的發送時間
5. 加入後續服務說明：
   - 說明我們會持續追蹤
   - 提供多種聯絡方式

請讓客戶感受到我們對他們的重視和專業服務。
```

## 特殊情境處理

### 情境 1：處理投訴或抱怨

```
請幫我寫一封回覆客戶投訴的 Email：

1. 客戶投訴內容：[描述投訴內容]
2. 我們的立場：[說明我們的立場和處理方式]
3. 道歉（如果需要）：真誠地道歉
4. 解決方案：具體說明我們會如何解決
5. 預防措施：說明我們會如何避免類似問題

請用專業、誠懇、積極的語氣，展現我們對問題的重視和解決的決心。
```

### 情境 2：拒絕請求

```
請幫我寫一封拒絕客戶請求的 Email：

1. 客戶請求：[描述請求內容]
2. 拒絕原因：[說明為什麼無法滿足，但要委婉]
3. 替代方案：[提供可能的替代方案]
4. 未來合作：表達未來合作的意願

請用專業、友善、建設性的語氣，不要讓客戶感到被拒絕，而是引導到其他可能性。
```

### 情境 3：緊急通知

```
請幫我寫一封緊急通知的 Email：

1. 緊急情況：[描述緊急情況]
2. 影響範圍：[說明影響範圍]
3. 應對措施：[說明我們採取的應對措施]
4. 後續更新：說明何時會提供更新資訊

請用清晰、直接、專業的語氣，讓收件人快速了解情況和應對方式。
```

## 使用技巧

### 1. 提供足夠的上下文

**不好的 Prompt**：
```
幫我寫一封 email
```

**好的 Prompt**：
```
請幫我寫一封會議邀請的 Email，收件人是合作 3 年的客戶專案經理。
我們要討論下週三下午 2:00 的專案進度會議，地點在我們公司 3 樓會議室。
這位客戶很重視效率，所以語氣要專業但簡潔，不要太多客套話。
```

### 2. 明確指出需要改進的地方

**不好的回饋**：
```
這個語氣不對
```

**好的回饋**：
```
這封 Email 的語氣太正式了，這位客戶是我們長期合作的夥伴，
希望語氣更親切一些，像是朋友之間的溝通，但還是要保持專業。
另外，開頭可以提到上次會議的內容，讓它更有個人化的感覺。
```

### 3. 逐步迭代，不要一次要求太多

**不好的方式**：
```
幫我寫一封完美的 email，要專業、親切、有說服力、包含所有細節、
符合公司規範、個人化、還要處理文化差異...
```

**好的方式**：
```
階段 1：先寫基本架構和主要內容
階段 2：加入具體的時間、地點、議程等細節
階段 3：調整語氣，讓它更親切
階段 4：加入個人化的元素和公司資訊
階段 5：根據收件人進行最後調整
```
