# 企劃撰寫 - 迭代 Prompt 參考

> **重要提示**：本教學使用迭代方式，從簡單的企劃架構開始，逐步完善成專業的企劃書。

## 使用情境

企劃撰寫是辦公室常見的工作，包括：
- 專案企劃書
- 活動規劃
- 提案文件
- 預算規劃
- 執行計畫

透過迭代方式，您可以：
- 快速建立企劃架構
- 逐步補充內容和細節
- 加入數據和證據支持
- 優化表達和說服力

## 迭代階段 1：基礎版本

### 目標
建立企劃的基本架構和核心內容。

### Prompt 範例

#### 情境 A：活動企劃

```
請幫我建立一個活動企劃的基本架構，活動主題是「新產品發表會」：

1. 活動目的：為什麼要辦這個活動
2. 目標對象：誰會參加這個活動
3. 活動時間和地點：基本資訊
4. 活動內容：大致要做什麼
5. 預期成果：希望達到什麼效果

請用簡潔的方式呈現，先建立基本框架即可。
```

#### 情境 B：專案企劃

```
請幫我建立一個專案企劃的基本架構，專案名稱是「客戶服務系統優化」：

1. 專案背景：為什麼要做這個專案
2. 專案目標：要達成什麼目標
3. 專案範圍：包含哪些工作
4. 執行時程：大概需要多久
5. 預期效益：完成後會有什麼好處

請先建立基本框架，細節後續再補充。
```

#### 情境 C：行銷企劃

```
請幫我建立一個行銷企劃的基本架構，產品是「新的手機 App」：

1. 行銷目標：要達成什麼行銷目標
2. 目標客群：要賣給誰
3. 行銷策略：大致要用什麼方式行銷
4. 預算範圍：大概需要多少預算
5. 預期效果：希望達到什麼效果

請先建立基本框架即可。
```

## 迭代階段 2：增加細節

### 目標
在基礎架構上補充具體細節，讓企劃更完整。

### Prompt 範例

#### 延續情境 A：活動企劃

```
請在剛才的活動企劃基礎上，增加以下細節：

1. 活動目的細節：
   - 主要目的：向目標客戶介紹新產品
   - 次要目的：收集客戶反饋、建立品牌形象
   - 預期參與人數：100-150 人

2. 目標對象細節：
   - 主要對象：現有客戶中的決策者
   - 次要對象：潛在客戶和媒體
   - 對象特徵：對新科技有興趣、有購買決策權

3. 活動內容細節：
   - 開場：公司高層致詞（15 分鐘）
   - 產品介紹：產品經理簡報（30 分鐘）
   - 產品體驗：現場體驗區（60 分鐘）
   - 問答時間：Q&A 環節（15 分鐘）
   - 交流時間：Networking（30 分鐘）

4. 活動時間和地點細節：
   - 日期：2024 年 3 月 15 日（週五）
   - 時間：下午 2:00-5:00
   - 地點：台北國際會議中心 3 樓
   - 交通資訊：提供捷運、停車資訊

5. 預期成果細節：
   - 至少 50 位客戶表達購買意願
   - 收集 30 份以上的客戶反饋問卷
   - 獲得 5 家以上媒體報導

請用結構化的方式呈現，讓企劃更完整。
```

#### 延續情境 B：專案企劃

```
請在剛才的專案企劃基礎上，增加以下細節：

1. 專案背景細節：
   - 現況問題：客戶服務系統回應速度慢、功能不足
   - 影響範圍：影響 80% 的客戶服務流程
   - 緊急程度：中等，但影響客戶滿意度

2. 專案目標細節：
   - 主要目標：提升客戶服務效率 50%
   - 次要目標：增加客戶滿意度、減少人力成本
   - 成功指標：回應時間從 24 小時縮短到 8 小時

3. 專案範圍細節：
   - 系統優化：升級現有系統、新增自動回覆功能
   - 流程改善：重新設計服務流程
   - 人員訓練：培訓客服人員使用新系統
   - 測試與上線：系統測試、逐步上線

4. 執行時程細節：
   - 第一階段（1-2 月）：需求分析與系統設計
   - 第二階段（3-4 月）：系統開發與測試
   - 第三階段（5 月）：人員訓練與試運行
   - 第四階段（6 月）：正式上線與優化

5. 預期效益細節：
   - 效率提升：客戶服務回應時間減少 50%
   - 成本節省：減少 30% 的人力成本
   - 客戶滿意度：從 70% 提升到 90%

請用條列式和表格的方式呈現，讓資訊更清晰。
```

## 迭代階段 3：優化表達

### 目標
改善語言表達和結構，讓企劃更有說服力。

### Prompt 範例

#### 延續情境 A：活動企劃

```
請優化剛才的活動企劃，讓它：

1. 更有說服力：
   - 強調活動的獨特性和價值
   - 說明為什麼現在是舉辦活動的最佳時機
   - 展現活動對參與者的好處

2. 結構更清晰：
   - 使用明確的標題和子標題
   - 用表格呈現時間表
   - 用條列式呈現重點

3. 語言更專業：
   - 使用專業的企劃術語
   - 避免口語化表達
   - 保持客觀但積極的語氣

4. 加入視覺元素說明：
   - 說明需要什麼視覺設計
   - 說明場地佈置需求
   - 說明需要什麼宣傳素材

5. 強化行動呼籲：
   - 明確說明需要什麼資源支持
   - 說明需要哪些部門配合
   - 說明下一步行動

請讓這份企劃看起來專業且具有說服力。
```

#### 延續情境 B：專案企劃

```
請優化剛才的專案企劃，讓它：

1. 更有邏輯性：
   - 清楚說明問題與解決方案的關聯
   - 說明為什麼這個方案是最佳選擇
   - 展現專案的可行性

2. 結構更專業：
   - 使用標準的專案企劃格式
   - 加入專案管理術語
   - 使用甘特圖或時程表（用文字描述）

3. 語言更精準：
   - 使用具體的數字和指標
   - 避免模糊的表述
   - 用數據支持論點

4. 加入風險評估：
   - 說明可能遇到的風險
   - 提出風險應對方案
   - 說明備援計畫

5. 強化執行可行性：
   - 說明需要的資源和人力
   - 說明技術可行性
   - 說明預算合理性

請讓這份企劃看起來專業、可行、有說服力。
```

## 迭代階段 4：專業化

### 目標
加入專業術語、數據支持、標準格式，符合專業企劃書標準。

### Prompt 範例

#### 延續情境 A：活動企劃

```
請將剛才的活動企劃專業化：

1. 使用標準企劃書格式：
   - 封面：活動名稱、日期、主辦單位
   - 目錄：各章節索引
   - 執行摘要：1-2 頁的企劃重點
   - 詳細內容：各章節詳細說明
   - 附錄：相關資料和參考文件

2. 加入數據支持：
   - 市場分析數據
   - 類似活動的成功案例
   - 預期 ROI（投資報酬率）
   - 參與人數預估的依據

3. 使用專業術語：
   - 用「目標客群」而不是「要邀請的人」
   - 用「執行時程」而不是「時間安排」
   - 用「預算分配」而不是「要花多少錢」

4. 加入詳細的執行計畫：
   - 工作分解結構（WBS）
   - 責任分工表（RACI）
   - 時程甘特圖（用文字描述）
   - 預算明細表

5. 加入評估機制：
   - 成功指標（KPI）
   - 評估方法
   - 後續追蹤計畫

請讓這份企劃符合專業企劃書的標準。
```

#### 延續情境 B：專案企劃

```
請將剛才的專案企劃專業化：

1. 使用標準專案企劃格式：
   - 專案概述
   - 專案背景與動機
   - 專案目標與範圍
   - 專案時程與里程碑
   - 專案資源與預算
   - 風險管理計畫
   - 品質管理計畫
   - 溝通管理計畫

2. 加入詳細的專案管理元素：
   - 專案組織架構
   - 角色與職責定義
   - 溝通計畫
   - 變更管理流程
   - 問題追蹤機制

3. 使用專業的專案管理術語：
   - 用「里程碑」而不是「階段」
   - 用「可交付成果」而不是「要完成的東西」
   - 用「風險緩解策略」而不是「應對方法」

4. 加入數據和證據：
   - 現況分析數據
   - 市場研究數據
   - 類似專案的成功案例
   - ROI 分析
   - 成本效益分析

5. 加入詳細的執行計畫：
   - 工作分解結構（WBS）
   - 時程規劃（用甘特圖文字描述）
   - 資源需求分析
   - 預算明細

請讓這份企劃符合專業專案企劃書的標準。
```

## 迭代階段 5：個人化

### 目標
根據公司文化、決策者偏好、實際需求進行最後調整。

### Prompt 範例

#### 延續情境 A：活動企劃

```
請將剛才的活動企劃個人化：

1. 根據公司文化調整：
   - 如果公司文化較創新，強調活動的創新元素
   - 如果公司文化較保守，強調活動的穩健執行
   - 符合公司的品牌形象和價值觀

2. 根據決策者偏好調整：
   - 如果決策者重視數據，加強數據分析部分
   - 如果決策者重視創意，加強創意和視覺設計部分
   - 如果決策者重視執行，加強執行細節部分

3. 根據實際資源調整：
   - 根據實際預算調整活動規模
   - 根據實際人力調整執行方式
   - 根據實際時間調整時程安排

4. 加入個人或團隊特色：
   - 展現企劃者的專業和創意
   - 加入過去的成功經驗
   - 展現對活動的熱情和投入

5. 考慮實際執行細節：
   - 加入實際可能遇到的問題和解決方案
   - 加入實際的聯絡窗口和負責人
   - 加入實際的備援計畫

請讓這份企劃既專業又符合實際需求。
```

#### 延續情境 B：專案企劃

```
請將剛才的專案企劃個人化：

1. 根據公司策略調整：
   - 對齊公司的年度目標
   - 符合公司的發展方向
   - 支援公司的核心業務

2. 根據決策者關注點調整：
   - 如果重視 ROI，加強投資報酬分析
   - 如果重視風險，加強風險管理部分
   - 如果重視執行，加強執行細節部分

3. 根據實際情況調整：
   - 根據實際預算調整專案規模
   - 根據實際人力調整專案時程
   - 根據實際技術能力調整專案範圍

4. 加入團隊優勢：
   - 展現團隊的專業能力
   - 加入過去的成功案例
   - 展現對專案的承諾

5. 考慮實際執行：
   - 加入實際的資源需求
   - 加入實際的風險和應對
   - 加入實際的溝通和協調機制

請讓這份企劃既專業又可行。
```

## 特殊情境處理

### 情境 1：預算有限的企劃

```
請幫我調整剛才的企劃，在預算有限的情況下：

1. 預算限制：[說明預算限制]
2. 優先順序：哪些項目是必須的，哪些可以調整
3. 替代方案：用更經濟的方式達成目標
4. 分階段執行：是否可以分階段執行，先做最重要的部分

請保持企劃的完整性，但用更經濟的方式呈現。
```

### 情境 2：時間緊迫的企劃

```
請幫我調整剛才的企劃，在時間緊迫的情況下：

1. 時間限制：[說明時間限制]
2. 快速執行方案：如何快速啟動和執行
3. 精簡流程：哪些流程可以簡化
4. 資源集中：如何集中資源在關鍵項目上

請保持企劃的品質，但用更快速的方式執行。
```

### 情境 3：高風險或創新的企劃

```
請幫我調整剛才的企劃，這是一個高風險或創新的企劃：

1. 風險說明：清楚說明風險和挑戰
2. 風險緩解：提出詳細的風險應對方案
3. 試點計畫：是否可以先做小規模試點
4. 成功案例：引用類似的成功案例
5. 備援方案：如果失敗的備援計畫

請讓決策者了解風險，但也看到機會和價值。
```

## 使用技巧

### 1. 提供足夠的上下文

**不好的 Prompt**：
```
幫我寫一個企劃
```

**好的 Prompt**：
```
請幫我建立一個新產品發表會的活動企劃。
我們公司是科技公司，目標客戶是中小企業主。
預算大約 50 萬，時間是下個月。
希望透過這個活動提升品牌形象並獲得潛在客戶。
```

### 2. 明確指出需要改進的地方

**不好的回饋**：
```
這個企劃不夠詳細
```

**好的回饋**：
```
這個企劃的活動內容部分還不夠詳細，請補充：
1. 每個環節的具體流程
2. 需要準備的物料和設備
3. 人員分工和職責
4. 時間安排的細節
```

### 3. 逐步迭代，不要一次要求太多

**不好的方式**：
```
幫我寫一個完美的企劃，要專業、詳細、有說服力、包含所有細節、
有數據支持、符合公司文化、還要考慮實際執行...
```

**好的方式**：
```
階段 1：先建立基本架構和核心內容
階段 2：補充具體細節和執行方式
階段 3：優化表達和結構
階段 4：加入專業格式和數據支持
階段 5：根據實際需求進行最後調整
```
