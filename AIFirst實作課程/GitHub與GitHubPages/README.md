# GitHub 與 GitHub Pages

## 課堂規劃（約）

- 註冊／登入 **GitHub**，建立新儲存庫，理解 README、檔案上傳與基本版控概念。
- 將前一階段（或本課產出）的網頁／專案檔案上傳至儲存庫，必要時用資料夾分類。
- 啟用 **GitHub Pages**（或依課程指定分支／資料夾），讓靜態網站可對外網址存取。
- 自訂首頁與導覽，把多個小作品連結成**個人作品集**入口（可持續擴充）。
- （選修）在本機安裝 Git 客戶端，練習 `clone`／`add`／`commit`／`push` 的最小流程。

## 作品（產出）

上傳程式碼並發布**個人作品集網站**

## 實作請放此處

- 可放靜態網站原始碼（如 `index.html`、資源檔）或子模組。
- 建議註明：儲存庫網址、GitHub Pages 發布網址。
