# 本專案技術架構說明

這個「管理者登入-登出流程」的專案結合了目前最流行的前端工具、後端即服務 (BaaS) 以及 AI 輔助開發平台。以下是這個專案所使用的核心技術說明：

## 1. 前端開發：React + Vite

*   **React**: 由 Meta (Facebook) 維護的開源 JavaScript 函式庫，專門用來建構使用者介面 (UI)。在這個專案中，React 負責管理網頁的畫面（例如：首頁、登入表單、管理者專屬畫面）以及元件的狀態（例如：使用者是否已登入、是不是管理者）。
*   **Vite**: 新一代的前端建置工具。相較於傳統的 Webpack，Vite 提供了極快的冷啟動速度和即時模組熱更新 (HMR)，讓開發 React 應用程式的體驗大幅提升。

## 2. 後端即服務 (BaaS)：Supabase

Supabase 是一個開源的 Firebase 替代方案，底層基於強大的 PostgreSQL 資料庫。我們在這個專案中使用了它的幾項核心功能：

*   **Supabase Auth**: 處理使用者身份驗證。它負責管理使用者的註冊、登入與登出流程（本範例使用 Email/密碼登入），讓我們不需要自己從頭寫一套安全的會員系統。
*   **PostgreSQL 資料庫**: 用來儲存應用程式的資料。我們在這裡建立了一張 `profiles` 資料表，用來記錄使用者的權限角色（例如：是 `user` 還是 `admin`）。
*   **RLS (Row Level Security)**: 資料列級別的安全性控制。這是 PostgreSQL 的一項強大功能，Supabase 將其完美整合。我們透過 RLS 政策 (Policies) 來限制誰可以讀取或寫入哪一筆資料，確保資料庫層級的絕對安全。

## 3. AI 輔助開發：Google AI Studio

*   **Google AI Studio (Build mode)**: 這是 Google 提供給開發者的 AI 工具。我們透過自然語言（Prompt）描述需求（例如：「幫我建立一個有登入按鈕和權限判斷的 React 網站」），AI 就能快速生成完整的專案程式碼架構。這大幅縮短了原型開發 (Prototyping) 的時間。
