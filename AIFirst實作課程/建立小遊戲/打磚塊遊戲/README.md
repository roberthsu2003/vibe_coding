# 打磚塊遊戲 - Prompt 參考

> **重要提示**：本專案使用 Vite、React 和 TypeScript 開發。建議學員先安裝 Node.js 環境（並使用 `npm create vite@latest` 建立專案），或直接使用線上編輯器（如 CodeSandbox、StackBlitz）來執行專案。

## 遊戲簡介
打磚塊（Breakout）是一個經典的街機遊戲。玩家控制一個平台（paddle）來反彈球，擊破上方的磚塊。目標是清除所有磚塊。

## 迭代階段 1：基礎版本

### Prompt 範例
```
請幫我建立一個打磚塊遊戲，使用 Vite、React 和 TypeScript。

基本需求：
1. 建立一個 React 元件，包含一個遊戲畫布區域（例如 800x600 像素）
2. 在 React 中使用 HTML5 Canvas 搭配 useRef 來繪製遊戲畫面
3. 在底部建立一個可移動的平台（paddle），使用 useEffect 監聽左右方向鍵或滑鼠移動來控制
4. 建立一個球，從平台上方開始移動
5. 實作遊戲迴圈 (Game Loop)：使用 requestAnimationFrame 搭配 useRef 來維持流暢的更新，避免觸發不必要的 React 重新渲染
6. 處理碰撞檢測：球碰到牆壁、平台、或磚塊時會反彈
7. 在頂部建立一排或多排磚塊 (使用二維陣列或物件陣列儲存狀態)
8. 當球擊中磚塊時，更新磚塊狀態使其消失
9. 當球掉到底部時，遊戲結束（或失去一條生命）

請使用清晰的程式碼結構，分離邏輯層與渲染層，並加上 TypeScript 型別與註解說明。
```

## 迭代階段 3：增加不同類型的磚塊

### Prompt 範例
```
請在打磚塊遊戲中增加不同類型的磚塊，並定義相關的 TypeScript Interface：

1. 普通磚塊：擊中一次就消失
2. 堅固磚塊：需要擊中 2-3 次才會消失 (需加上生命值屬性)
3. 特殊磚塊 (包含掉落道具功能)：
   - 加分磚塊：擊中後掉落金幣，接到獲得額外分數
   - 加速磚塊：擊中後球的速度增加
   - 擴展磚塊：擊中後掉落道具，接到後平台變寬
   - 多球磚塊：擊中後產生額外 2 顆球 (需將球的狀態改為陣列管理)

2. 用不同顏色或圖案 (Canvas 繪製邏輯) 來區分不同類型的磚塊和損壞程度

3. 特殊效果有持續時間限制：使用 setTimeout 或在迴圈中計時，時間到後恢復原狀（例如平台擴展效果持續 10 秒）
```

## 技術建議

- **環境建置**：建議使用 `npm create vite@latest game -- --template react-ts` 快速建立專案。
- **渲染與效能分離**：UI 介面（分數、生命、開始按鈕）使用 **React State (`useState`)** 渲染；而遊戲畫布、球的座標、磚塊狀態等高頻率變動的資料，強烈建議使用 **`useRef`** 儲存，並完全透過 **HTML5 Canvas** 搭配 **`requestAnimationFrame`** 繪製，以避免 React 頻繁重新渲染導致卡頓。
- **碰撞數學**：使用基本的幾何運算（AABB 碰撞檢測）來處理球與磚塊、平台的碰撞及反彈角度邏輯。
- **資料持久化**：使用 **`localStorage`** 來儲存最高分或關卡解鎖進度。

## 學習重點

1. **階段1**：學習 React `useRef` 與 Canvas 的完美結合、實作不依賴 React Render 的高效能 Game Loop、基礎物理反彈與 AABB 碰撞演算法。
2. **階段2**：學習 React UI 狀態與 Canvas 遊戲狀態的溝通與整合、遊戲生命週期管理。
3. **階段3**：學習 TypeScript Interface 設計（定義複雜的磚塊屬性與道具）、陣列/多物件狀態更新邏輯。
4. **階段4**：學習資料驅動設計 (Data-driven design)、使用 JSON/陣列結構定義關卡地圖。
5. **階段5**：學習 Canvas 粒子系統實作、Web Audio API 整合、視覺特效優化。
