# 反應力測試遊戲 - Prompt 參考

> **重要提示**：本專案使用 Vite、React 和 TypeScript 開發。建議學員先安裝 Node.js 環境（並使用 `npm create vite@latest` 建立專案），或直接使用線上編輯器（如 CodeSandbox、StackBlitz）來執行專案。

## 遊戲簡介
反應力測試遊戲是一個測試玩家反應速度的遊戲。玩家需要在看到特定訊號時快速做出反應（例如按下按鍵或點擊滑鼠）。

## 迭代階段 1：基礎版本

### Prompt 範例
```
請幫我建立一個反應力測試遊戲，使用 Vite、React 和 TypeScript。

基本需求：
1. 建立一個 React 元件，包含一個滿版或固定大小的遊戲互動區域（例如 div 區塊）
2. 實作遊戲狀態機 (State Machine)，例如：'idle' (等待開始), 'waiting' (準備中，還不能按), 'ready' (現在按！), 'result' (顯示結果)
3. 使用 React 處理點擊或鍵盤事件，按下按鈕開始後，進入 'waiting' 狀態，並使用 setTimeout 隨機等待一段時間（例如 1-5 秒）
4. 等待時間結束後，狀態切換至 'ready'，改變畫面顏色或顯示「現在！」的提示
5. 玩家需要盡快點擊畫面或按下空白鍵
6. 若提早按下（'waiting' 狀態時），顯示「太早按了！」並重置
7. 使用 performance.now() 搭配 useRef 或 useState 計算反應時間（毫秒）
8. 顯示反應時間結果，並提供按鈕重新測試

請使用清晰的元件結構，並加上 TypeScript 型別與註解說明。
```

## 迭代階段 3：難度調整與挑戰模式

### Prompt 範例
```
請在反應力測試遊戲中增加難度調整和挑戰模式：

1. 難度選擇（透過狀態管理控制參數）：
   - 簡單：等待時間較長（3-7 秒），提示明顯（大字體、強烈對比色）
   - 中等：等待時間中等（1-5 秒），提示正常
   - 困難：等待時間較短（0.5-3 秒），提示較不明顯（顏色變化較小或目標較小）

2. 挑戰模式：
   - 連續測試模式：連續進行 5 次測試，結束後結算並計算平均反應時間
   - 極限模式：反應時間超過某個閾值（如 500ms）則直接判定失敗
   - 時間挑戰：實作一個 30 秒的倒數計時器，在時間內完成盡可能多的測試

3. 反應時間評級：
   - 根據反應時間給予評級（例如：極快 < 200ms、快 < 300ms、普通、慢）
   - 顯示評級徽章或對應的顏色提示
```

## 技術建議

- **環境建置**：建議使用 `npm create vite@latest game -- --template react-ts` 快速建立專案。
- **時間測量精度**：強烈建議使用 **`performance.now()`** 來精確測量時間。在 React 中，為了避免 State 更新造成的重新渲染延遲，記錄開始時間點時請使用 **`useRef`** 儲存。
- **狀態機設計**：遊戲流程（Idle -> Waiting -> Ready -> Result）非常適合用狀態機概念設計，可用 `useState` 或 `useReducer` 清楚管理。
- **資料儲存**：使用 **`localStorage`** 來持久化測試記錄。
- **圖表套件**：資料分析視覺化強烈建議安裝 React 專用的圖表套件，例如 **Recharts** 或 **react-chartjs-2**。

## 學習重點

1. **階段1**：學習精確時間測量 (`performance.now()`)、`useRef` 保存跨渲染變數、`useEffect` 與 `setTimeout` 清除機制 (Cleanup)。
2. **階段2**：學習陣列狀態管理、陣列方法 (map, reduce) 計算統計數據、元件 Props 傳遞。
3. **階段3**：學習更複雜的狀態控制 (挑戰模式邏輯)、條件渲染與元件共用設計。
4. **階段4**：學習 React 中的音效控制 (`Audio` API 與 `useRef`)、Tailwind/CSS 動畫整合。
5. **階段5**：學習第三方套件整合 (如圖表庫)、自訂 Hook 封裝、Blob 檔案下載與 JSON 處理。
