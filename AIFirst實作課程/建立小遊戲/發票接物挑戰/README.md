# 發票接物挑戰 (Invoice Catching Challenge)

## 遊戲介紹

這是一款以日常生活中收集發票為主題的趣味接物小遊戲！玩家將化身為發票收集達人，控制螢幕下方的籃子，在限時 40 秒內盡可能接住從天而降的各種發票來獲取高分。挑戰你的反應神經，收集越多環保發票分數越高！

### 🎮 玩法與計分規則

- ☁️ **雲端發票（藍色）**：環保最高分 **+3 分** 
- 📱 **電子發票（綠色）**：常見好夥伴 **+2 分**
- 🧾 **紙本收據（白色）**：基本小收據 **+1 分**
- 💣 **地雷炸彈（紅色）**：當心陷阱！接到會倒扣 **-5 分**

### 🕹️ 操作方式

- **電腦版**：使用鍵盤的「左」、「右」方向鍵，或是滑鼠操控籃子移動。
- **手機版**：直接透過手指左右滑動螢幕來靈活操控。

# 發票接物挑戰遊戲建立手冊 (Prompt Guide)

如果你想製作一個類似「發票接物挑戰」的網頁遊戲，你可以使用以下幾種不同階段的 Prompt 指令。這些指令是專門為 **Google AI Studio Build** 或類似的 AI 程式碼編輯器設計的。

---

## 🚀 階段一：建立遊戲核心 (基礎指令)

如果你剛開始，可以使用這段指令來建立遊戲的基本架構：

> **Prompt:**
> 請幫我建立一個名為「發票接物挑戰」的 React 遊戲。
> 1. **核心玩法**：玩家控制螢幕底部的籃子，接住上方掉落的物體。可以用左右方向鍵或滑鼠/手指滑動控制。
> 2. **物體種類**：
>    - 雲端發票（藍色）：+3分
>    - 電子發票（綠色）：+2分
>    - 紙本收據（白色）：+1分
>    - 炸彈（紅色）：-5分
> 3. **遊戲介面**：使用 Tailwind CSS 呈現簡潔現代的深色 UI，顯示目前分數、最高分和倒數計時（40秒）。
> 4. **技術要求**：使用 Vite + React，並使用 `motion` (motion/react) 處理動畫。

---

## 🎨 階段二：美化視覺與增加素材 (進階指令)

當基本功能完成後，你可以要求 AI 加入圖片和音效：

> **Prompt:**
> 我想優化遊戲的視覺與聽覺體驗：
> 1. **圖片資源**：請為各種發票類型和炸彈引用圖片（如果是本地資源請確保路徑正確，如果是外部連結請使用 Picsum 或 Unsplash 預留位）。
> 2. **音效設定**：加入背景音樂 (BGM) 以及接住物體時的音效。音效需要支援靜音切換功能。
> 3. **動畫效果**：當籃子接到發票時，要有微小的縮放動畫。炸彈爆炸時要有震動效果。
> 4. **響應式設計**：確保遊戲在電腦（鍵盤/滑鼠）和手機（觸控）上都能順暢遊玩。

---

## 🛠️ 階段三：專業部署與優化 (最後衝刺)

如果你想讓遊戲看起來更專業，並準備部署到 GitHub Pages：

> **Prompt:**
> 請幫我進行最後的優化：
> 1. **部署設定**：將 `vite.config.ts` 的 `base` 路徑設為 `'./'`，以便我部署到 GitHub Pages 而不會路徑出錯。
> 2. **遊戲平衡**：隨著遊戲進行，物體掉落速度要稍微加快，增加挑戰性。
> 3. **暫停功能**：按下 ESC 或點擊畫面上的按鈕可以暫停遊戲並顯示設定選單。
> 4. **說明視窗**：遊戲開始前顯示玩法說明彈窗。

---

## 💡 給使用者的建議小撇步 (Tips)

- **具體描述得分規則**：AI 很擅長處理邏輯，如果你明確告訴它「雲端發票是 3 分」，它會幫你寫好所有的條件判斷。
- **強調效能**：如果是在 Canvas 上繪製大量物體，可以提醒 AI「請使用 `requestAnimationFrame` 來優化性能」。
- **提到 Canvas vs DOM**：
    - 如果物體很多（幾百個），建議用 **Canvas**。
    - 如果物體不多，純 **DOM + Tailwind** 會比較容易調整樣式。
- **環境變數**：如果你需要用 Gemini AI 做更智慧的評語，記得提到 `GEMINI_API_KEY` 的使用。
