# API 產生器

## 課堂規劃（約）

- 選定一種簡單**資料模型**（例如待辦、書籍、商品），定義欄位與唯一 ID。
- 用 AI 產出 **REST 風格 CRUD API**（建立、讀取、更新、刪除），並約定 URL 與 HTTP 方法。
- 決定儲存方式：記憶體、JSON 檔、或輕量資料庫（依講義）；先求能跑再求完美。
- 撰寫或產生簡易 **API 說明**（端點、參數、範例請求）；用 curl 或 Thunder Client 測試。
- 討論錯誤處理（找不到資源、格式錯誤）與基本安全性（勿暴露敏感設定）。

## 作品（產出）

以 AI 建立資料的 **CRUD API**

- 測試方式（如 curl 範例或 API 文件連結）。
