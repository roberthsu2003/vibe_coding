# 自動化流程

## 課堂規劃（約）

- 定義輸入：一或多份**文字文件**（或長文），以及期望的報告格式（章節、長度、語氣）。
- 第一階段：**摘要**（萃取重點、關鍵決策）；第二階段：**彙整**（多文件合併成單一報告）。
- 若需「自動化」：用腳本、試算表外掛、或雲端工作流程（依講義工具）串起「讀檔 → 呼叫模型 → 寫出」。
- 設計批次情境：多檔同樣流程、或每週重跑；記錄失敗重試與人工檢查點。
- 注意 token 與檔案大小限制；長文件可分段摘要再合併（map-reduce 概念）。

## 作品（產出）

**自動摘要文件**並**彙整成報告**

## 實作請放此處

- 輸入文件範例、輸出報告範例。
- 自動化腳本或工作流程設定（依工具而定）。
