# 建立公司表單：辦公室飲料訂購系統

本單元將帶領學生利用 AI 建立一個與 Google Sheets 串接的實戰應用。學生將學會如何透過精確的 Prompt 引導 AI 產出前端介面、後端邏輯以及資料庫連接代碼。

## 課堂規劃

- **需求分析**：定義飲料訂購系統需要的欄位（品項、甜度冰塊、訂購人、金額）。
- **資料庫設計**：在 Google Sheets 建立 `Menu`（菜單）與 `Orders`（訂單）分頁。
- **AI 協作實作**：
    - 使用 Prompt 產生前端 HTML/CSS/JS。
    - 使用 Prompt 產生 Google Apps Script (GAS) 後端。
    - 串接測試與 Bug 調校。
- **自動化應用**：利用 AI 彙整今日訂單摘要，方便發送給飲料店家。

## 💡 Prompt 指令示範 (教學重點)

教學時，請引導學生按照以下階段下指令：

### 第一階段：設計前端介面
> **Prompt 範例**：
> 「我想要建立一個辦公室飲料訂購系統的前端網頁。請使用 HTML 和 Vanilla CSS 幫我寫一個精緻的表單，包含：訂購人姓名（文字框）、選擇飲料（下拉選單）、甜度（Radio Buttons）、冰塊（Radio Buttons）、數量（數字輸入框）。請讓設計看起來現代且具有專業感，並確保在手機上也能輕鬆點餐。」

### 第二階段：建立 GAS 後端邏輯
> **Prompt 範例**：
> 「我正在使用 Google Apps Script 作為後端。請幫我寫一個 GAS 腳本，包含兩個主要功能：
> 1. `doGet()`：從 Google Sheet 的 'Menu' 分頁讀取飲料名稱與價格，並以 JSON 格式回傳。
> 2. `doPost()`：接收前端傳來的訂單資料（姓名、飲料、甜度冰塊、數量、金額），並將其寫入 'Orders' 分頁的最後一行。
> 請確保包含適當的錯誤處理，並回傳成功訊息。」

### 第三階段：串接與整合
> **Prompt 範例**：
> 「我現在有了前端 HTML 和 GAS 的 URL。請幫我寫一段 JavaScript，在網頁載入時呼叫 GAS 的 doGet 取得菜單並動態填入下拉選單。另外，當使用者點擊『送出訂單』按鈕時，請計算總金額並透過 fetch POST 傳送到 GAS。請加入送出後的成功提示動畫。」

## 作品（產出）

- **實戰作品**：一個可以動態讀取菜單、提交訂單並寫入 Google Sheets 的 Web App。
- **技術文件**：記錄下你所使用的所有 Prompts 以及調校過程。
