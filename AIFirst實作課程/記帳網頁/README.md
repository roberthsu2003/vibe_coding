# 自然語言 ＋ 迭代的開發方法 (記帳 App 範例)

在 Vibe Coding 裡，你不用先寫規格書，也不用一次想清楚所有細節。只要**用自然語言說出目標**（例如：「開發一個簡易記帳 App」），讓 AI 產出**第一個原型**，再透過**迭代**一步步把應用程式做完善。

## 1. 開發流程：一句話 → 原型 → 迭代完善

### 第一步：用自然語言說出目標

不需要寫技術規格，用平常說話的方式描述你想做的東西就好。我們以「記帳 App」為例：

**原始生成 Prompt：**
```text
請用 vite-react-typescript 做一個單一網頁的記帳 App：
1. 可以輸入「支出名稱」與「金額」；
2. 有一個列表顯示所有的支出記錄；
3. 自動計算並顯示「總支出金額」；
4. 介面要簡潔現代。
請提供完整的 React 程式碼。
```

### 第二步：AI 產出第一個原型

AI 會根據你的描述，產出一版**可以跑、可以使用**的程式。這一版通常已經具備基本功能：輸入框、新增按鈕、記錄列表，以及自動加總的功能。

雖然這版還沒辦法永久儲存資料（重新整理就會消失），但重點是**先有東西可以看、可以操作**。

### 第三步：用迭代讓應用程式更完善（加入儲存與進階功能）

拿到第一版之後，我們開始透過對話引導 AI 補上重要的功能，例如你提到的**儲存紀錄**。

**迭代範例：**

1. **加入儲存功能（LocalStorage）：** 
   - 「請幫我加上 LocalStorage 功能，讓我在關閉網頁或重新整理後，之前的記帳記錄依然存在。」
2. **加入分類與日期：** 
   - 「在輸入時增加一個『分類』（例如：食物、交通、娛樂）的下拉選單，並自動帶入今天的日期。」
3. **優化視覺與體驗：** 
   - 「請使用 Tailwind CSS 讓介面變美，加入一點陰影感和圓角，並讓總金額顯示得更醒目。」
4. **加入圖表分析：** 
   - 「請使用 Recharts 或簡單的 CSS 圓餅圖，幫我分析不同分類的支出佔比。」

每完成一輪「描述 → AI 修改 → 你檢視」，應用程式就離你心中的樣子更近一步。這就是**迭代**。

---

## 💡 給學生的建議
- **從簡單開始**：不要第一句話就要求 AI 做完所有功能。先讓它做出「能輸入、能顯示」的版本。
- **提到儲存**：在 Web 專案中，如果你還沒學習後端資料庫，可以告訴 AI「請使用 **LocalStorage** 來儲存資料」，這樣資料就會留在瀏覽器裡。
