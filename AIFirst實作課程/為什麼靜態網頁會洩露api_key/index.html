⚠️ 靜態網頁 API Key 洩露示範

這個範例示範：直接在前端 JavaScript 放入 API Key 的危險性

🧪 Step 1：輸入你的 Gemini API Key 並提問

回應結果會顯示在這裡...

🔍 Step 2：打開開發者工具，看看 API Key 在哪裡 關鍵

  1. 按下 F12（或右鍵 → 檢查）開啟開發者工具
  2. 點選上方的 「Network（網路）」 分頁
  3. 重新點「送出問題」
  4. 點選出現的 generativelanguage.googleapis.com 請求
  5. 在 Headers（標頭） 或 Request URL 中，可以直接看到你的 API Key！
📌 你也可以在原始碼中找到 API Key：
右鍵 → 「檢視頁面原始碼」
使用 Ctrl+F 搜尋 API_KEY 或 AIzaSy，Key 就直接在原始碼裡！

💸 Step 3：洩露 API Key 的後果

  • 任何人只要「檢視原始碼」就能拿走你的 Key
  • 別人用你的 Key 大量呼叫 API → 你付費！
  • Google 可能因為濫用而停用你的帳號
  • Key 一旦公開，即使刪掉也可能已被爬蟲收走

✅ Step 4：正確做法 — 用後端保護 API Key

API Key 應該放在伺服器端（後端），前端只和你自己的後端溝通，後端再去呼叫 Gemini。 這樣使用者永遠看不到 Key。

// ❌ 錯誤：Key 直接寫在前端 JS const API_KEY = "AIzaSy..."; fetch(`https://generativelanguage.googleapis.com/...?key=${API_KEY}`, ...) // ✅ 正確：前端只呼叫自己的後端 API fetch("/api/ask", { method: "POST", body: JSON.stringify({ question: "..." }) }) // 後端（Vercel Function / Express）才持有 API Key
