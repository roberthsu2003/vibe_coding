# 自然語言 ＋ 迭代的開發方法

在 Vibe Coding 裡，你不用先寫規格書，也不用一次想清楚所有細節。只要**用自然語言說出目標**（例如：「開發一個雙班競賽計時器」），讓 AI 產出**第一個原型**，再透過**迭代**一步步把應用程式做完善。


## 1. 開發流程：一句話 → 原型 → 迭代完善

### 第一步：用自然語言說出目標

不需要寫技術規格，用平常說話的方式描述你想做的東西就好。我們以「雙班競賽計時器」為例：

**原始生成 Prompt：**
```text
請用 vite-react-typescript 做單一網頁：
1. 兩個獨立計時器，給兩班比賽同時計時；
2. 每組有開始、暫停、重設；
3. 時間用 MM:SS（或 HH:MM:SS）、字大易讀；
4. 兩組互不干擾；
5. 不需後端。
請給完整程式碼與簡短操作說明。
```

### 第二步：AI 產出第一個原型

AI 會根據你的描述，產出一版**可以跑、可以使用**的程式。這一版通常已經具備基本功能：兩個大大的計時器、控制按鈕，以及清楚的時間顯示。

這一版不必完美，重點是**先有東西可以看、可以操作**。

### 第三步：用迭代讓應用程式更完善

拿到第一版之後，不要一次要求「全部重寫」，而是**一次改一點**，用對話引導 AI 補上你想要的東西。

**迭代範例：**
1. **加入計分功能：** 「請在每個計時器旁邊加上『加分』與『減分』的按鈕，方便老師紀錄兩班的分數。」
2. **優化視覺與體驗：** 「請幫我加入深色模式，並讓按鈕看起來更現代化、大一點，方便在平板上點擊。」
3. **增加響鈴功能：** 「當計時器到達 00:00 時，請播放一個簡單的提示音，並讓螢幕閃爍紅色。」
4. **紀錄比賽歷史：** 「增加一個『紀錄結果』的功能，按下去後會將目前的班級比分紀錄在下方的表格中。」

每完成一輪「描述 → AI 修改 → 你檢視」，應用程式就離你心中的樣子更近一步。這就是**迭代**。
